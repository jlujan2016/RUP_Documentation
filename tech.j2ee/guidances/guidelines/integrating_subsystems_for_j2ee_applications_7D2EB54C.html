Directriz: Integración de subsistemas para aplicaciones J2EE
Esta directriz trata la integración de subsistemas para una aplicación J2EE.
Relaciones
Elementos relacionados
  • Integrar los subsistemas
Descripción principal

La integración de subsistemas de implementación para una aplicación J2EE supone el ensamblar los elementos de implementación individuales que forman el subsistema de implementación (por ejemplo, componentes J2EE y sus descriptores de despliegue iniciales) en un módulo J2EE (o conjunto de módulos J2EE). El módulo o módulos J2EE resultantes representan la compilación del subsistema de implementación. Consulte la sección Directriz: compilaciones para aplicaciones J2EE para obtener más información sobre las compilaciones para aplicaciones J2EE.

El ensamblaje de elementos de implementación individual puede requerir algún tipo de reconciliación entre estos elementos. Además, los resultados de la integración se recogen en el módulo o módulos resultantes y sus descriptores de despliegue. Consulte la sección Directriz: ensamblaje de módulos J2EE para obtener más información sobre la forma en que ensamblar módulos J2EE.

El módulo o módulos resultantes y su o sus descriptores de despliegue servirán como entrada para las pruebas de integración del subsistema.
