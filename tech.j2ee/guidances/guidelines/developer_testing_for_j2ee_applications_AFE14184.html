Directriz: Pruebas de desarrollador para aplicaciones J2EE
En esta directriz se describe algunas consideraciones adicionales que hay que tener en cuenta al realizar pruebas unitarias de componentes J2EE.

Descripción principal

Configuración del entorno de pruebas

Las clases Java normales se pueden compilar y probar sin crear un módulo J2EE. Sin embargo, componentes J2EE, tales como EJB y servlets, se basan en servicios que proporciona el contenedor, y por lo tanto se deben ensamblar y desplegar en un contenedor para poder probarlos.

Por lo tanto, el implementador de los componentes J2EE debe ensamblarlos en un módulo o módulos J2EE y posiblemente en una aplicación J2EE y, a continuación, desplegar estos archivadores en un entorno de pruebas unitarias antes de realizar dichas pruebas unitarias.

Consulte la sección Directriz: ensamblaje de módulos J2EE para obtener más información sobre la forma en que ensamblar módulos J2EE.

Consulte la sección Directriz: ensamblaje de aplicaciones J2EE para obtener más información sobre la forma en que ensamblar aplicaciones J2EE.

Consulte la sección Directriz: despliegue de módulos y aplicaciones J2EE para obtener más información sobre la forma en que desplegar aplicaciones J2EE.

Directrices generales en relación a las pruebas en J2EE

Además de probar la funcionalidad que proporciona el código fuente, para EJB, JSP y servlets, el contenedor en sí mismo proporciona una funcionalidad que se necesita probar como, por ejemplo, la relacionada con la persistencia, las transacciones y la seguridad.
