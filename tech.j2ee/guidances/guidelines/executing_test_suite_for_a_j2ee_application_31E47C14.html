Directriz: Ejecución de un conjunto de pruebas para una aplicación J2EE
En esta directriz se describen algunas consideraciones adicionales que hay que tener en cuenta al realizar pruebas de integración de componentes J2EE.

Descripción principal

Pruebas de integración de subsistemas

Para las pruebas de integración de subsistemas, la compilación a probar (empaquetada en uno o más módulos J2EE) se debe ensamblar en una aplicación J2EE y, a continuación, se debe desplegar en el entorno de pruebas.

Para obtener más información sobre las compilaciones de aplicaciones J2EE, consulte la sección Directriz: compilaciones para aplicaciones J2EE. .

Consulte la sección Directriz: ensamblaje de aplicaciones J2EE para obtener más información sobre la forma en que ensamblar aplicaciones J2EE.

Consulte la sección Directriz: despliegue de módulos y aplicaciones J2EE para obtener más información sobre la forma en que desplegar aplicaciones J2EE.

Pruebas de integración de sistemas

Para las pruebas de integración de sistemas, la compilación a probar (empaquetada en una aplicación J2EE) se debe desplegar en un entorno de pruebas.

Para obtener más información sobre las compilaciones de aplicaciones J2EE, consulte la sección Directriz: compilaciones para aplicaciones J2EE. .

Consulte la sección Directriz: despliegue de módulos y aplicaciones J2EE para obtener más información sobre la forma en que desplegar aplicaciones J2EE.
