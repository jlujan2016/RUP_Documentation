Directriz: Compilaciones para aplicaciones J2EE
En esta directriz se trata de las compilaciones en el contexto del desarrollo de aplicaciones J2EE.

Elementos relacionados
  • Compilación
Descripción principal

La definición de compilación se proporciona en la sección Producto de trabajo: compilación, y se puede resumir como una "versión operativa de un sistema o parte de un sistema".

En un proyecto J2EE, una compilación a nivel de sistema está formada habitualmente de una aplicación J2EE (consulte la sección Directriz: aplicación J2EE), pero también puede incluir:

  • Una o más bases de datos.
  • Aplicaciones que no son de J2EE.
  • Módulos J2EE autónomos (consulte la sección Directriz: aplicación J2EE para obtener más información).

Una compilación a nivel de subsistema habitualmente está formada por uno o más módulos J2EE, pero también puede encapsular bases de datos así como otros archivos.

Las compilaciones se crean mediante la integración de componentes. En un entorno J2EE, normalmente supone ensamblar módulos J2EE y aplicaciones J2EE. Consulte la sección Directriz: ensamblaje de módulos J2EE y la sección Directriz: ensamblaje de aplicaciones J2EE para obtener más información.
