Instrucciones de la herramienta: Ejecución de pruebas de desarrollador de J2EE mediante Rational XDE Developer - Java Platform Edition
En esta directriz de herramientas se describe la forma en que el desarrollador puede probar las aplicaciones J2EE mediante Rational XDE.
Herramienta: Rational XDE Developer
Descripción principal

Objetivo

En esta directriz de herramientas se describe la forma en que el desarrollador puede probar las aplicaciones J2EE mediante Rational XDE.

Visión general

En esta guía de herramientas se llevan a cabo los siguientes pasos:

  • Configuración del entorno de pruebas.
  • Ejecución de las pruebas.

Los enlaces a los temas de la ayuda en línea de Rational XDE™ están marcados con .

Configuración del entorno de pruebas

En una aplicación J2EE, para poder realizar una prueba unitaria en un único componente, este componente se debe haber ensamblado en una aplicación J2EE y estar realmente desplegado. Desde luego, en este punto hay que realizar un mínimo ensamblaje y despliegue, tan sólo el suficiente como para tener en ejecución el componente en el entorno de pruebas. Consulte la sección Guía de herramientas: ensamblaje de módulos y aplicaciones mediante Rational XDE Developer - Java Platform Edition para obtener más información sobre el ensamblaje de aplicaciones J2EE. Consulte la sección Guía de herramientas: despliegue de módulos y aplicaciones J2EE mediante Rational XDE Developer - Java Platform Edition para obtener más información sobre la forma en que desplegar aplicaciones J2EE.

Si se utiliza WebSphere Studio Application Developer, los componentes se pueden probar en su entorno de pruebas de WebSphere (WTE) incorporado. Tan sólo hay que seleccionar los componentes en el navegador de J2EE en la perspectiva de J2EE e invocar el mandato de menú de contexto "Ejecutar en servidor". Consulte la ayuda en línea de WebSphere Studio Application Developer para obtener más información.

Para depurar una aplicación en un servidor de aplicaciones distinto, hay que iniciar el servidor con la depuración habilitada. Consulte la documentación del servidor para saber cómo hacerlo. A continuación, una vez se despliega la aplicación, se puede conectar el depurador de Eclipse al servidor para depurar la aplicación. Observe que la mayoría de los servidores de aplicación no proporcionan una forma para depurar las JSP (que se compilan en código Java). Consulte la documentación del servidor.

Ejecución de las pruebas

Una vez esté desplegada la aplicación, se pueden depurar los componentes como normalmente se haría, utilizando el depurador de Java incorporado y cualquier otra herramienta de análisis y depuración que esté disponible.

Observe que si se utiliza WebSphere Studio Application Developer y XDE, existe la posibilidad de utilizar la característica Trace de Rational Purify y XDE que permite depurar y rastrear componentes J2EE en ejecución en el entorno de pruebas de WebSphere (WTE) que es parte de Application Developer. Observe que XDE Trace no da soporte a componentes que se ejecuten en servidores de aplicaciones J2EE remotos. Consulte la sección Análisis del comportamiento de tiempo de ejecución con Rational PurifyPlus para obtener más información. Además, actualmente no es posible utilizar Trace o PurifyPlus con páginas JSP.
