Concepto: Tipo de producto de trabajo
Esta categoría estándar representa una agrupación de productos de trabajo relacionados que, a diferencia del dominio, está más orientada a la presentación.
Descripción principal

Los productos de trabajo pueden adoptar diversas formas, como:

  • Un modelo, como el modelo de caso de uso o el modelo de diseño, que contiene otros artefactos.
  • Un elemento de modelo, es decir, un elemento de un modelo, como una clase de diseño, un caso de uso o un subsistema de diseño.
  • Datos de proyecto que pueden guardarse en bases de datos u otros tipos de depósitos de información tabular como las hojas de cálculo.
  • El código fuente y los programas ejecutables que contribuyen al producto o solución
  • Diversos tipos de documentos, por ejemplo, un documento de especificación, como la especificación de requisitos de software, o un documento de plan, como el plan de requisitos de software.

Pueden por tanto categorizarse en consecuencia. Un ejemplo es "Especificación", que categoriza las especificaciones de requisitos que definen un sistema con límites bien definidos, como los casos de uso o la especificación de requisitos funcionales. A diferencia de los dominios, un único producto de trabajo puede categorizarse en distintos tipos de productos de trabajo.
