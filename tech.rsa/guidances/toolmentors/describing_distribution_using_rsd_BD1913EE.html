Descripción principal

Visión general

En esta guía de la herramienta se llevan a cabo los pasos siguientes:

  • Analizar los requisitos de distribución
  • Definir la configuración de red
  • Asignar elementos del sistema a nodos  

Analizar los requisitos de distribución

No existe ninguna guía específica de herramienta para este paso; sin embargo, a continuación se presenta una lista de sugerencias que pueden ser de utilidad a la hora de documentar la información relacionada con la distribución:

  • Considere dividir los diagramas grandes en varios diagramas más pequeños para direccionar un subconjunto coherente del aspecto de distribución.
  • Utilice vistas de <<perspectiva>>, ya que pueden ser útiles para comunicar información de distribución específica.
  • Utilice notas y color como efectos visuales para llamar la atención de las funciones importantes representadas en los diagramas.
  • Proporcione detalles coherentes con el nivel de abstracción necesario. Exponga sólo los elementos que sean esenciales para la compresión:
    • Utilice Suprimir de Diagrama para eliminar los elementos que no desee. Este mandato suprimirá los elementos del diagrama pero no tendrá ningún impacto en el modelo subyacente.
    • Utilice filtros para ocultar/mostrar detalles. Para acceder a este mandato, pulse el botón derecho del ratón sobre elementos de modelo en un diagrama y seleccione Filtros.

Definir la configuración de red

Perfeccione el diagrama de despliegue creado en Tarea: Análisis de arquitectura. El documento técnico Directrices de estructura de modelo de RSx recomienda la utilización de un modelo en blanco como punto de inicio.

Asignar elementos del sistema a nodos

Anote el diagrama de despliegue o añada diagramas adicionales para mostrar el despliegue lógico (clases, subsistemas o hebras de control correlacionados con nodos) y el despliegue físico (artefacto correlacionados con nodos). Los pasos son los siguientes:

  1. Añada diagramas de despliegue adicionales, según se requiera. Consulte Adición de elementos de diagrama en la ayuda en línea.
  2. Añada nodos o instancias de nodo al diagrama. Consulte Nodos, Instancias de nodo en la ayuda en línea.
  3. Arrastre y suelte elementos de modelo de diseño o elementos de modelo de implementación en el diagrama. Si el elemento que va a desplegar aún no se ha modelado, añádalo según sea necesario al modelo adecuado. (Consulte Directrices de estructura de modelo para RSx para saber donde se modela cada elemento.)
  4. Si los elementos que se añaden son artefactos, añada directamente relaciones de despliegue entre nodos y elementos desplegados.
  5. En el caso de elementos lógicos, existe un número de opciones de representación:
    • Se podría utilizar una asociación general para mostrar el despliegue lógico.
    • Un artefacto que se enlaza a través de una relación de manifestación con elementos lógicos que implementa y se relaciona con un nodo utilizando una relación de despliegue directa.
