Descripción principal

Visión general

En esta guía de la herramienta se llevan a cabo los pasos siguientes:

  • Identificar oportunidades de reutilización
  • Ingeniería inversa de componentes y bases de datos
  • Actualizar la organización del modelo de diseño
  • Actualizar la vista lógica

Información adicional sobre la herramienta

Identificar oportunidades de reutilización

Busque componentes o subsistemas existentes que ofrezcan interfaces similares. Es posible que se hayan empaquetado algunos activos para su reutilización deliberada utilizando RAS (especificación de activo reutilizable). Si se han creado con la herramienta, dichos activos se pueden examinar y cargar. Consulte Empaquetado de activos para su reutilización en la ayuda en línea.

Ingeniería inversa de componentes y bases de datos

Nota: algunas de las capacidades de la herramienta mencionadas en esta sección no se soportan en RSM.

Para los activos que no incluyen un modelo, es posible que le resulte útil revertir la ingeniería de un modelo con el objeto de comprender mejor el diseño. Algunos de estos componentes se pueden descubrir mediante la visualización de código y las funciones de detección de patrón/antipatrón que forman parte del soporte del análisis de la arquitectura. Si se demuestra que el activo puede volver a utilizarse, los diagramas servirán de base para crear o generar elementos de modelo nuevos y el activo se considerará apto para ser candidato RAS. Consulte las directrices Descubrimiento, análisis y control de arquitectura.

Para un activo que controla la empresa, es posible que existan oportunidades de realizar cambios poco importantes en una interfaz candidata que mejoren su conformidad con la interfaz deseada. Para los activos que no controla la empresa, puede crear elementos de diseño de puente o adaptador que correlacionen la interfaz deseada con interfaces que proporcionen los activos reutilizados. Consulte Identificar elementos de diseño para conocer las directrices sobre la creación y modificación de interfaces y clases.

Actualizar la organización del modelo de diseño

Reorganice los elementos de diseño en paquetes, según sea necesario.  Consulte la documentación técnica: Directrices de estructura de modelo para RSx para conocer más directrices sobre la organización del modelo de diseño.

Actualizar la vista lógica

Si existe una vista lógica separada, se debe mantener. Consulte la documentación técnica Directrices de estructura de modelo para RSx.
