Producto de trabajo: Configuración de entorno de prueba
Este producto de trabajo especifica una disposición de hardware, software y la configuración del entorno asociado que son necesarios para habilitar que se efectúen pruebas precisas que evaluarán uno o más elementos del destino de la prueba.
Objetivo

Para ayudar a  garantizar que:

  • Los resultados de las actividades de prueba son precisas y válidas,
  • Las actividades de prueba  tienen una probabilidad más alta de reproducirse sistemáticamente

Descripción
Esquematización breve

Cada configuración de entorno de prueba debe considerar varios aspectos que incluyen los siguientes:

  • Los requisitos de hardware de sistema básicos; por ejemplo, procesadores, almacenamiento de memoria, almacenamiento de disco duro, dispositivos de interfaz de entrada/salida
  • El entorno de software subyacente básico; por ejemplo, sistema operativo y herramientas de productividad básicas como correo electrónico, sistema de calendario, etc.
  • Hardware periférico adicional especializado de entrada/salida; por ejemplo, escáneres de códigos de barras, impresoras de recibos, cajas registradoras, dispositivos sensores, etc.
  • El software necesario para el hardware periférico de entrada/salida especializado; por ejemplo, software de controladores, interfaz y pasarela
  • El conjunto mínimo de herramientas de software necesarias para facilitar la prueba, la evaluación y las actividades de diagnóstico; por ejemplo, diagnósticos de memoria, ejecución de prueba automatizada, etc.
  • Los valores de configuración necesarios para las opciones de hardware y software; por ejemplo, resolución de la pantalla de vídeo, asignación de recursos, variables de entorno, etc.
  • Los consumibles "preexistentes" necesarios; por ejemplo, conjuntos de datos rellenados, dockets de impresora de visualización, y similares.
Descripción principal

Cada configuración del entorno de prueba especifica un valor apropiado y controlado en que se realizará la prueba y las actividades de evaluación necesarias. Al proporcionar un entorno controlado construido a partir de una configuración conocida donde realizar estas actividades, se garantiza que los resultados de estos esfuerzos son precisos, válidos, y tienen posibilidades más elevadas de reproducirse sistemáticamente. Un entorno de prueba bien controlado es un aspecto importante de la resolución del análisis de anomalías y de errores.

Opciones de representación

Siempre que sea posible, intente mantener el número total de configuraciones de entorno de prueba exclusivas al mínimo y a un  nivel manejable. Esto puede conseguirse consolidando entornos similares, normalmente cuando se utilizan perfiles de software y hardware base similares en los que sólo existen pequeñas diferencias en los valores de configuración. Sin embargo, tenga cuidado en no consolidar las configuraciones de entorno de prueba hasta el punto de invalidar la integridad y el propósito de cada configuración.

Recomendamos utilizar herramientas de creación de imágenes de disco duro que permitan realizar copias de seguridad de las configuraciones de entorno de prueba y restaurarlas fácilmente. Considere también la posibilidad de utilizar unidades de disco duro extraíbles o intercambiables para seguir mejorando la precisión y la eficacia.

Opcionalmente, la configuración de entorno de prueba puede incluirse en el plan de desarrollo de software.
