Tarea: Planificar la integración del subsistema
En esta tarea se describe cómo planificar el orden en el que se integrarán los elementos contenidos en un subsistema de implementación.

Pasos
Definir las compilaciones

Estudie los guiones de uso y los casos de ejemplo que se han seleccionado para la iteración actual. Seleccione uno o varios casos de ejemplo que serán el objetivo de cada incremento de la integración. Puede que tenga que seleccionar sólo una parte de un caso de ejemplo que afecte a este subsistema.

Capture el plan para integrar el subsistema, ya sea en el plan de compilación de integración del proyecto o en un plan de compilación de integración local en el subsistema.

Identificar las clases

Identifique las clases que participen en los casos de ejemplo seleccionados. Cada caso de ejemplo se describe en los diagramas de secuencia, los diagramas de comunicación o los diagramas de clase de una realización del guión de uso de diseño. Identifique qué clases debe implementar y qué clases se han implementado ya. Identifique también las clases que no participan en el caso de ejemplo, pero que son necesarias como fragmentos para simulación.

Las clases se identifican a partir de las realizaciones de guión de uso de diseño.

Actualizar las importaciones del subsistema

Identifique qué otros subsistemas de implementación son necesarios para esta compilación. Decida qué versión va a utilizar de cada subsistema. Actualice las dependencias de importación de este subsistema a las versiones correctas de los otros subsistemas.

Si se han promocionado recientemente nuevas líneas base del sistema, el integrador también tendrá que decidir cuándo se debe actualizar el espacio de trabajo de integración del subsistema (recrear las líneas base). Esta decisión se basa en qué fase del ciclo de desarrollo se encuentra. Si el desarrollo del subsistema es inestable en algunas áreas críticas, deberá posponer la recreación de las líneas base.

Al final del proyecto, cerca del release (ya sea interno o externo), es crucial que los subsistemas tengan conjuntos de importación consistentes. Es entonces cuando hay una mayor urgencia de mantener actualizadas las líneas base.
