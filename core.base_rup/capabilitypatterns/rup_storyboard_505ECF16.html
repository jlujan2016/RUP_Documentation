Producto de trabajo: Guión gráfico
Este artefacto es una descripción lógica y conceptual de la funcionalidad del sistema para un caso de ejemplo específico, incluyendo la interacción necesaria entre los usuarios del sistema y el sistema. Un guión gráfico "cuenta una historia específica".
Objetivo

Las personas siguientes utilizan los guiones gráficos:

  • analistas de sistemas, para explorar, clarificar y capturar la interacción de comportamiento prevista por el usuario como parte de la adquisición de requisitos.
  • diseñadores de interfaz de usuario, para diseñar la interfaz de usuario y para construir un prototipo de la interfaz de usuario;
  • diseñadores de las clases que proporcionan funcionalidad a la interfaz de usuario. Utilizan esta información para comprender las interacciones del sistema con el usuario, para que puedan diseñar adecuadamente las clases que implementará la interfaz de usuario;
  • quiénes diseñan la versión siguiente del sistema para comprender cómo lleva a cabo el sistema el flujo de sucesos;
  • los verificadores para probar las características del sistema;
  • el gestor para planificar y supervisar el análisis y el trabajo de diseño.

Es importante recordar que el objetivo principal de los guiones gráficos es comprender el flujo y las interacciones globales, no crear un prototipo ni probar el aspecto de la interfaz de usuario. El guión gráfico no debe cubrir los objetos gráficos de la interfaz de usuario y otras preocupaciones de la interfaz de usuario (esto debería cubrirse en el prototipo de interfaz de usuario).

Descripción principal

Los guiones gráficos dan soporte a la adquisición de requisitos proporcionante importantes mecanismos de información de retorno para descubrir requisitos desconocidos o poco claros.

Factores clave
Puede definirse un guión gráfico para cada guión de uso, de esta manera se da soporte a un enfoque de la ingeniería de software controlado por guiones de uso, además de proporcionar un medio excelente para validar las expectativas del usuario (actor) de estos guiones de uso y su rol en los flujos de sucesos de guiones de uso.
Personalización
Opciones de representación

Los guiones gráficos se pueden expresar con representaciones visuales o textuales, o una combinación de ambas.

Decida si los guiones gráficos son útiles para su proyecto. El contenido de los guiones gráficos se debe personalizar para que dé soporte a las necesidades del proyecto. 

Los guiones gráficos a menudo se consideran productos de trabajo transitorio, y pueden dejarse sin mantenimiento una vez se han comprendido los requisitos de comportamiento y el proyecto está en marcha para agilizar la creación de prototipos o la implementación de la interfaz de usuario. No obstante, en algunos casos, puede ser interesante mantener los guiones gráficos durante unas cuantas iteraciones, por ejemplo si hay requisitos complejos en relación con la interfaz de usuario para los que se necesita tiempo (durante varias iteraciones) para comprenderlos. Asimismo, los guiones gráficos, junto con la propia interfaz de usuario, son una entrada útil para la documentación del usuario final.
