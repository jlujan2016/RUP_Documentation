Producto de trabajo: Paquete de diseño
Este producto de trabajo es una recopilación de clases, relaciones, ejecuciones de guión de uso, diagramas y otros paquetes. Se utiliza para estructurar el modelo de diseño dividiéndolo en componentes más pequeños.
Objetivo

Los paquetes de diseño se utilizan para agrupar los elementos de modelo de diseño relacionados con objetivos organizativos y, a menudo, para la gestión de la configuración. A diferencia del Producto de trabajo: Subsistema de diseño, un paquete de diseño no ofrece una interfaz formal, aunque puede exponer algunos de sus contenidos (marcados como 'públicos') que ofrecen comportamiento. Los paquetes de diseño deben utilizarse principalmente como herramienta organizativa del modelo, para agrupar cosas relacionadas; si la semántica de comportamiento es necesaria, utilice los subsistemas de diseño.

Un paquete de diseño y su contenido son responsabilidad de un único Rol: Diseñador. Los elementos dentro del paquete pueden ser dependientes de los elementos contenidos por otros paquetes; esto ocasiona las dependencias entre paquetes. Las dependencias de paquete se pueden utilizar como herramienta para analizar la elasticidad del modelo de diseño: un modelo con paquetes de dependencia cruzada es menos elástico al cambio.

Opciones de representaciónRepresentación UML: Paquete en el modelo de diseño. 

Los paquetes se utilizan en los modelos para agrupar elementos de modelo similares, mejorando la organización del modelo y facilitando su comprensión. En los modelos grandes, el empaquetado es fundamental. Incluso en modelos más pequeños, un empaquetado adecuado puede mejorar sensiblemente la inteligibilidad del modelo. Casi siempre es útil hacer algún tipo de empaquetado. Para obtener más información, consulte el apartado Directriz de producto de trabajo: Paquete de diseño.
