Producto de trabajo: Atributos de requisitos
Este producto de trabajo describe un repositorio de requisitos, atributos y dependencias del proyecto para ayudar a gestionar el cambio desde la perspectiva de los requisitos.
Objetivo

El producto de trabajo Atributos de requisitos proporciona un repositorio del texto de requisito, atributos y rastreabilidad para todos los requisitos. Debe ser accesible para toda la empresa de desarrollo.

Descripción
Esquematización breve

Las vistas siguientes deben estar disponibles para ver el estado actual del producto de trabajo:

1. Matrices de atributos de requisitos

1.1 <tipo de requisito>
Para cada tipo de requisito, presente una matriz que liste los requisitos en un eje y todos los atributos en el otro eje. Para cada requisito, muestre el estado de sus atributos respectivos.

2. Matrices de rastreabilidad de requisitos

2.1 <tipo de requisito>

2.1.1 <tipo de requisito rastreado>
Para cada tipo de requisito, presente una matriz que liste los requisitos en un eje y todos los elementos rastreados en el otro eje. Para cada rastreo, muestre su estado (Correcto o sospechoso).

3. Árbol de rastreabilidad de requisitos

3.1 <tipo de requisito>

3.1.1 <tipo de requisito rastreado>
Un árbol de rastreabilidad proporciona una vista gráfica de las relaciones de rastreabilidad con los requisitos de un tipo de requisito específico (la raíz).

Opciones de representación

Decida si los atributos deben documentarse en los productos de trabajo de requisitos, en el plan de gestión de requisitos, o gestionarse utilizando herramientas de gestión de requisitos como, por ejemplo, Rational RequisitePro.  Si utiliza una herramienta de gestión de requisitos, puede mantener la información del repositorio directamente en las vistas guardadas de la base de datos de gestión de requisitos.
