Producto de trabajo: Plan de compilación de integración
Este artefacto proporciona un plan detallado para su integración dentro de una iteración.
Objetivo

El objetivo del plan de compilación de integración es definir el orden en que los componentes deben implementarse, qué construcciones deben crearse cuando se integra el sistema, y cómo deben valorarse.

Las personas siguientes utilizarán el plan de compilación de integración:

  • Implementadores, para planear el orden en que implementar los elementos de diseño, y qué y cuándo entregar la integración del sistema
  • Integrador, como herramienta del plan
  • Diseñador de prueba, para definir las pruebas para la iteración

Factores clave

Debe ajustar la esquematización del plan de compilación de integración para que se adecue a la naturaleza del proyecto. Por ejemplo, si el sistema es grande, puede ser necesario tener planes subsidiarios para cada subsistema de implementación. La formalidad y el alcance del material de prueba contenido o al que se hace referencia desde el plan de compilación de integración variará dependiendo del significado de la construcción. Evidentemente, la compilación final de una iteración se valorará formalmente contra todos los criterios de evaluación descritos en el plan de iteración.
