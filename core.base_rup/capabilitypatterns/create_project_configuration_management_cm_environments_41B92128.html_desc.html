Descripción

Este trabajo se realiza al garantizar que los artefactos esenciales están a disposición de los desarrolladores e integradores en los distintos espacios de trabajo públicos y privados a medida que se necesitan, y se les proporciona una línea base y un almacenamiento adecuados para su uso posterior. La configuración del entorno de CM supone la creación de la estructura de directorios del producto, los depósitos, los espacios de trabajo (desarrollo e integración) y la asignación de los recursos de máquina (servidores y espacio en disco).

Personal

Para configurar un entorno adecuado, una persona que desempeñe el rol de gestor de configuración debe conocer ampliamente las estructuras del producto global y debe trabajar estrechamente con el arquitecto de software para garantizar que se han establecido los elementos de marca de posición de CM adecuados.

Una persona que desempeñe el rol de integrador en este trabajo debe asegurarse de que los artefactos entregados desde los espacios de trabajo de desarrollo hayan sido suficientemente probados para que puedan incorporarse a una compilación que se pueda probar. El rol de integrador debe estar familiarizado con las políticas, las compilaciones y las prácticas de prueba de CM del proyecto.

Utilización
Instrucciones de utilización

Normalmente, este trabajo empieza en la fase inicial y continua durante todo el ciclo vital. En general, este trabajo es más predominante en las iteraciones justo al principio de cada fase (o al final de cada fase anterior) para preparar un entorno de CM adecuado para la fase próxima. Observe que los requisitos de un entorno CM a menudo cambian de una fase a otra a medida que el tamaño del equipo se amplía o se reduce: un equipo de desarrollo más reducido en las fases iniciales normalmente requiere un entorno menos formal del que se requiere en fases posteriores.

Este trabajo no se considera opcional, aunque la cultura del proyecto hará que este trabajo sea una pequeña consideración o un gran cometido. Observe que, si hay un entorno CM existente, este trabajo puede centrarse más en adaptar dicho entorno para adecuarlo al proyecto o a la fase del ciclo vital.
