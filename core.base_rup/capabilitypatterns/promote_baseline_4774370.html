Tarea: Promocionar las líneas base
En esta tarea se define cuándo ha finalizado la creación de una línea base y cómo debe etiquetarse una línea base finalizada.
Objetivo
  • El objetivo de esta tarea es garantizar que las líneas base (componentes probados individualmente de diversos implementadores y equipos de desarrollo, combinados entre sí para trabajar conjuntamente como un solo producto) están "etiquetados" para reflejar el nivel de madurez, estabilidad y calidad del software que hayan podido alcanzar. La versión de línea base, una vez etiquetada correctamente, está disponible para release o continuar el desarrollo en iteraciones subsiguientes

Pasos
Decidir la etiqueta apropiada para la línea base

Las líneas base ayudan a mantener sincronizado el equipo del proyecto. Proporcionan una vista de la versión más reciente de los activos del proyecto. Como tales, las líneas base deben crearse periódicamente de acuerdo con las políticas de CM del proyecto.

Las líneas base pueden denominarse a partir de la fase y de la iteración en que se han creado. En este ejemplo, el nombre de una línea base podría ser BL-Product-X-c2, lo que implica que es la línea base creada al final de la segunda iteración de la fase de construcción.

Sin embargo, otros convenios de etiquetado para líneas base podrían corresponderse con el nivel de prueba, y calidad, que una línea base del producto haya podido alcanzar. En este caso, por ejemplo, la línea base podría etiquetarse de la siguiente manera:

  • Integración probada
  • Sistema probado
  • Aceptación probada
  • Producción

El convenio de etiquetado descrito más arriba sugiere que una vez que una línea base se ha probado y verificado, para alcanzar un determinado de calidad se promociona y se etiqueta con una etiqueta de orden superior.
