Tarea: Verificar la instalación y configuración de herramientas
En esta tarea se describe cómo verificar que la configuración de desarrollo está preparada para ser utilizada por el proyecto.
Objetivo

El objetivo de esta tarea es verificar que las herramientas pueden utilizarse para desarrollar el sistema.

Descripción principal

Las herramientas y la infraestructura de desarrollo deben verificarse antes de que el proyecto empiece a utilizarlas. La forma de hacerlo variará claramente en función de las habilidades, la tecnología y las herramientas.

Pasos
Verificar el entorno

Verifique si el entorno contiene el hardware, el software y los datos correctos. Verifique si se ha instalado el hardware correcto. Puede hacerlo visualmente o mediante una herramienta (como por ejemplo utilizando Propiedades de Windows para Mi PC).  

Verificar las herramientas

Verifique si se ha instalado la configuración correcta de software. Para ello, examine los valores del registro, los archivos 'ini' o lance la herramienta y examine algunas opciones de información u opciones de configuración.

Verificar los datos

Verifique que los datos contengan los datos apropiados. La verificación de datos puede exigir la utilización de herramientas para inspeccionar visualmente los datos o la utilización de una aplicación para visualizar los datos. En algún momento, uno o varios casos de uso pueden seleccionarse y ejecutarse (uno o más casos de ejemplo) para cada herramienta con el fin de garantizar que la herramienta y los resultados de utilizar la herramienta son coherentes con la necesidad.

Ejecutar las herramientas

Reúna un pequeño grupo de personas que conozcan bien las herramientas y el proceso de desarrollo  y deje que ejecuten las herramientas. 

  • Pruebe muchos usuarios en conflicto de varios sitios
  • Se ha ejecutado al menos un caso de ejemplo de caso de uso para cada herramienta para verificar la correcta instalación y configuración de las herramientas.  
  • Pruebe los casos de ejemplo normales del proceso de desarrollo y en las directrices de las herramientas.  
  • Pruebe la integración entre las diferentes herramientas. 

Emita solicitudes de cambio si es necesario.
