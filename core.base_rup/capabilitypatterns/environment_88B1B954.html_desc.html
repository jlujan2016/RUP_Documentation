Patrón de posibilidad: Entorno
Este patrón de posibilidad se encarga de las actividades y el flujo de trabajo de la disciplina de entorno.
DescripciónEstructura de desglose de trabajoAsignación de equiposUtilización del producto de trabajo
Objetivo
La finalidad de las actividades del entorno es proporcionar a la empresa de desarrollo de software un entorno de desarrollo de software, los procesos y las herramientas que den soporte al equipo de desarrollo.
Relaciones
Contexto
  • Classic RUP (for large projects)
Descripción
Durante las iteraciones iniciales del proyecto, se prepara el entorno específico del proyecto. El entorno específico del proyecto se perfecciona durante cada iteración, según sea necesario.

Utilización
Notas de utilización

Decida cómo llevar a cabo el flujo de trabajo

Las decisiones siguientes deben tomarse considerando el flujo de trabajo de la disciplina de entorno:

  • Decida cómo llevar a cabo el flujo de trabajo observando las actividades de este flujo de trabajo. Estudie el diagrama con sus condiciones de vigilancia y las directrices. Decida las actividades que se van a realizar y su orden.  
  • Decida las partes de las actividades de entorno que deben llevarse a cabo. En general, los artefactos de la disciplina de entorno se introducen a medida que se necesitan. Por ejemplo, una guía de estilo de manuales sólo se desarrolla si el proyecto va a desarrollar materiales de soporte para el usuario.
  • Decida en qué punto del ciclo vital del proyecto desea introducir cada parte del flujo de trabajo. El proceso de desarrollo se introduce siempre al principio del proyecto.  Si lo desea, puede utilizar el guión de desarrollo para documentar las decisiones detalladas de personalización específicas del proyecto. Un proceso de entrega personalizado puede importarse en Rational Portfolio Manager para ayudar a asignar trabajo y hacer seguimiento del progreso del proyecto. Los otros artefactos se introducen cuando se necesitan.
