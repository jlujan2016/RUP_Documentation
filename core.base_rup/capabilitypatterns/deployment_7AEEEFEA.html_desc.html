Descripción

Para explicar el trabajo que supone la disciplina de despliegue, las actividades y productos de trabajo se organizan en un patrón de posibilidad para la disciplina.

El despliegue consiste en hacer que el producto de software esté disponible ara el usuario final y es la culminación del esfuerzo de desarrollo de software. 

La planificación del despliegue (Actividad: Planificar el despliegue) se inicia al principio de ciclo vital del proyecto y aborda no sólo la producción del entregable de software que se va a entregar, sino también el desarrollo del material de formación y el material de soporte para el sistema, para asegurarse de que los usuarios finales pueden utilizar satisfactoriamente el producto de software que se entrega.

El material de soporte (Actividad: Desarrollar material de soporte) cubre toda la información que precisará el usuario final para instalar, operar, utilizar y mantener el sistema que se entrega. También incluye material de formación para todas las posiciones distintas que se precisarán a la hora de hacer un uso eficaz del nuevo sistema.

La disciplina de despliegue pone un gran énfasis en asegurarse de que el producto se ha probado adecuadamente antes de su release a la base de clientes. La actividad Gestionar la prueba de aceptación cubre los dos tipos de entornos de prueba. En primer lugar, la compilación debe probarse suficientemente en el entorno de prueba de desarrollo (Actividad: Gestionar la prueba de aceptación) y, a continuación, volver a probarse en la ubicación de destino (Actividad: Gestionar la prueba de aceptación para una instalación personalizada). El 'entorno de prueba' debe ser una 'instancia' del entorno de prueba. 

Una vez que el producto se ha probado en el sitio de desarrollo, debe prepararse para su entrega al cliente. El release puede crearse para una prueba de versión beta, para un despliegue de prueba para los usuarios finales o, dependiendo de su nivel de madurez, para el producto final. En la Actividad: Producir la unidad de despliegue se describe la logística de la creación del release de un producto que consta del software y de los necesarios productos de trabajo adjuntos que se precisan para instalarlo eficazmente y utilizarlo. 

Un programa de versión beta hace referencia al proceso empleado por una empresa para solicitar información de retorno de un subconjunto de usuarios acerca de productos que se están desarrollando. La información de retorno se utiliza para mejorar el producto. En la Actividad: Producto de prueba de versión beta se describen las actividades que posibilitan el despliegue iterativo de un producto y el compromiso sistemático con el cliente a la hora de crear el producto final.

En cuanto al software 'comercializable', en la Actividad Empaquetar el producto se describen la actividades necesarias para tomar el producto de software, los scripts de instalación y los manuales de usuario, y empaquetarlos para su producción masiva como el resto de productos de consumo.

Puede solicitar al contratista de desarrollo que le instale el software o bien puede adquirir el software en un establecimiento o descargarlo de Internet. En la Actividad: Proporcionar acceso a un sitio de descargas se hace referencia al producto que se ofrece para la compra y descarga a través de Internet como un canal de distribución de software.

Decida cómo llevar a cabo el flujo de trabajo

Las decisiones siguientes deben tomarse considerando el flujo de trabajo de la disciplina de despliegue:

  • Decida cómo llevar a cabo el flujo de trabajo observando las actividades de este flujo de trabajo. Estudie el diagrama con sus condiciones de vigilancia y directrices. Decida las actividades que se van a realizar y su orden. La decisión más importante que debe tomar es el tipo de despliegue que desea llevar a cabo:
    • Instalación personalizada
    • Oferta de producto 'comercializable'
    • Acceso al software a través de Internet
  • Decida las partes de las actividades de despliegue que deben llevarse a cabo. A continuación, se indican algunas partes que son más o menos optativas y pueden introducirse de forma relativamente independiente del resto.
