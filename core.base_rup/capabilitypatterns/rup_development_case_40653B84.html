Producto de trabajo: Guión de desarrollo
Este artefacto describe el proceso de desarrollo que ha elegido para seguir en el proyecto.
Objetivo

El objetivo del guión de desarrollo es capturar el proceso personalizado para una organización o para un proyecto individual. Describe cómo se ha configurado el proceso.

Descripción
Esquematización breve

El guión de desarrollo incluye información sobre las disciplinas de RUP que está utilizando la organización/proyecto.

Normalmente contiene la siguiente información específica de la organización/proyecto:

  • Información sobre cómo utilizar los productos de trabajo para cada disciplina. Por ejemplo, referencias a las plantillas que la organización/proyecto debe utilizar para describir los productos de trabajo.
  • Referencias a directrices e información que la organización/proyecto desea utilizar además de la información contenida en RUP.
  • ` Referencias al contenido de RUP en lugar de repetir esta información en el guión de desarrollo.

En general, el guión de desarrollo debe hacer referencia al proceso de desarrollo de software y a cualquier activo del proceso específico de la organización/proyecto, como por ejemplo directrices, plantillas, etc., para obtener información detallada, en lugar de repetir la información, incorporada.

El guión de desarrollo también incluye información sobre lo que se ha personalizado y por qué.


Descripción principal

El guión de desarrollo describe el proceso de desarrollo que debe utilizarse para una organización o para un proyecto. Describe cómo se aplica el proceso en el contexto de la organización/proyecto y cómo se ha configurado el proceso para la organización/proyecto.  El guión de desarrollo se utiliza para capturar aspectos del proceso que son específicos de la organización o del proyecto. Proporciona una visión general del proceso que debe seguirse y todas las personas que participan en el proyectos deberían entenderlo.

Factores clave

Dependiendo del nivel de personalización que se esté ejecutando (consulte los niveles de personalización definidos en el apartado Concepto: Personalización de RUP), el guión de desarrollo puede utilizarse como:

  • Vehículo para la personalización del proceso (es su objetivo principal y para el que se ha utilizado históricamente).
  • Una herramienta de planificación para preparar una personalización más extensa utilizando Rational Method Composer.

A continuación, el guión de desarrollo puede incluirse como parte del propio proceso de desarrollo  o simplemente archivarse como documentación histórica de lo que se ha añadido/eliminado/cambiado del proceso original.  El guión de desarrollo generalmente se incluye como parte del proceso de desarrollo en aquellos casos en que el proceso que se esta personalizando no puede cambiarse, o bien cuando hay información que debe capturarse en relación con el proceso que no puede capturarse en el propio proceso.
