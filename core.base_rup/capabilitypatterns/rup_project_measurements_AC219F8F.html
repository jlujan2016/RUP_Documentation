Producto de trabajo: Medidas del proyecto
Este artefacto es el repositorio activo de datos de métricas del proyecto. Contiene las medidas más recientes del proyecto, recursos, proceso y producto en el nivel primitivo y derivado.
Objetivo

El producto de trabajo Medidas del proyecto proporciona el almacenamiento para los datos de métricas del proyecto. Se actualiza a medida que se realizan medidas o cuando estas están disponibles. También contiene las métricas derivadas que se calculan a partir de los datos primitivos y también deben almacenar información (procedimientos y algoritmos, por ejemplo) sobre cómo se obtienen las métricas derivadas. Los informes sobre el estado del proyecto, por ejemplo, del progreso hacia objetivos (funcionalidad, calidad, etc.), gastos, y otros consumos de recursos, se generan mediante las medidas del proyecto (consulte el apartado Producto de trabajo: Valoración de estado). Las visualizaciones más frecuentes, o incluso aparentemente continuas, del estado del proyecto son posibles utilizando herramientas donde los agentes de recopilación de datos de software automatizados obtienen visualizaciones en tiempo real del estado del proyecto.

Descripción principal

El formato y el contenido de las medidas del proyecto dependen de las métricas seleccionadas y de la tecnología utilizada para la recopilación y el almacenamiento. Es esencialmente una base de datos de asociaciones métrica-valor e información aliada para su recopilación y cálculo. La forma puede ser tan simple como un conjunto de archivos mantenidos manualmente por el gestor de proyectos.  Sin embargo, recomendamos que la recopilación y el almacenamiento se automaticen y, siempre que sea posible, no sean intrusivos.

Opciones de representación

En proyectos pequeños, las medidas del proyecto sólo pueden existir en forma de informes del sistema de seguimiento de defectos y de una hoja de cálculo para realizar un seguimiento del progreso. En proyectos más grandes o más formales, puede haber una amplia selección de métricas que se gestionan utilizando una o varias bases de datos. Puede ser el caso de un producto de trabajo distribuido.  Por ejemplo, las diversas métricas que ha seleccionado el gestor de proyectos pueden producirse mediante varias herramientas distintas, y las tareas de recopilación e informe se realizan manualmente.  Otro ejemplo: el progreso del proyecto se puede notificar desde un plan del proyecto que el gestor de proyectos actualiza de forma rutinaria a partir de la información de estado que proporcionan los miembros del equipo en hojas de cálculo.
