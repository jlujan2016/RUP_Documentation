Tarea: Definir la lista de materiales
En esta tarea se describe la lista de materiales, un inventario del software y de los materiales que deben entregarse como parte del producto global.
Objetivo
  • Crear una lista completa de los productos de trabajo que componen la compilación/producto. La lista incluye elementos de configuración de software, documentación y scripts de instalación. En el caso de productos empaquetados, la lista de materiales debe identificar los elementos de material gráfico y los elementos de empaquetado que componen el producto final.

Descripción principal

La lista de materiales sirve como inventario del software y de los materiales que deben entregarse como parte del producto global. La lista de materiales indica los componentes que constituyen una versión determinada de un producto, y dónde se pueden encontrar los componentes físicos. Describe los cambios que se efectúan en la versión y hace referencia a cómo se puede instalar el producto. 

La lista de materiales debe actualizarse para cada compilación y, evidentemente, para cada iteración. El gestor de despliegue debe asegurarse de que se llevan a cabo los pasos siguientes.

Pasos
Listar todos los elementos entregables
Objetivo Garantizar que la lista de materiales cumple los requisitos del proyecto global 

El gestor de despliegue debe asegurarse de que todos los elementos necesarios contractualmente para la aceptación del producto están listados y contabilizados. El plan de desarrollo de software como artefacto global y compuesto que recopila toda la información necesaria para gestionar el proyecto es una buena fuente de lo que se necesitará y desarrollará durante la realización del proyecto.

El gestor de despliegue debe contar con el plan de aceptación del producto para describir cómo evaluará el cliente los entregables y determinar si éstos cumplen el conjunto predefinido de criterios de aceptación. La lista de materiales debe tener en cuenta todos elementos necesarios para la aceptación del producto.

Desde un punto de vista táctico, el plan de iteración y el plan de creación de integración son una buena fuente para determinar lo que se tiene que desarrollar para una determinada iteración. 

Mantener la lista de materiales
Objetivo Garantizar que el proyecto tiene una lista actualizada de los productos de trabajo que componen la compilación del producto. 

Todos los elementos que entran a forman parte de una compilación deben aparecer en la lista de materiales. La lista de materiales debe actualizarse para cada compilación sucesiva y luego tomarse como línea base para la revisión al final de una iteración.
