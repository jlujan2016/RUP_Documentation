Tarea: Desarrollar el plan de gestión de requisitos
En esta tarea se describe cómo desarrollar un plan para documentar requisitos, sus atributos y directrices para la rastreabilidad y gestión de los requisitos del producto.
Objetivo

El objetivo de esta tarea es desarrollar un plan de gestión de requisitos que especifique la información y los mecanismos de control que se recopilarán y utilizarán para medir, informar y controlar los cambios efectuados en los requisitos del producto.

Descripción principal
Antes de empezar a describir los requisitos del proyecto, debe decidir cómo documentarlos y organizarlos, y también cómo utilizar los atributos de requisitos al gestionar los requisitos durante todo el ciclo de vida del proyecto.

Elegir los atributos apropiados y la rastreabilidad adecuada para los requisitos del proyecto le ayudará a:

  • Valorar el impacto en un proyecto de un cambio en un requisito
  • Valorar el impacto de un error de una prueba en los requisitos
  • Gestionar el ámbito del proyecto
  • Verificar que la implementación satisface todos los requisitos del sistema.
  • Verificar que la aplicación sólo hace las actividades para las que fue diseñada.
  • Gestionar los cambios.

Documente todas las decisiones relativas a documentos de requisitos, elementos de rastreabilidad (consulte los apartados Rastreabilidad y Tipos de requisitos), directrices y estrategias para los atributos de requisitos en el plan de gestión de requisitos.

Pasos
Establecer la rastreabilidad

En primer lugar, debe identificar los elementos de rastreabilidad entre los que desea establecer enlaces de rastreabilidad. Los elementos de rastreabilidad más importantes, y la rastreabilidad típica entre ellos, se describen en el apartado Concepto: Rastreabilidad.

El resultado se documenta en un conjunto de matrices de rastreabilidad de requisitos, que forman parte del producto de trabajo atributos de requisitos.

Elegir los atributos de requisitos

Los atributos de requisitos se utilizan para realizar el seguimiento de la información asociada con un elemento de rastreabilidad, habitualmente para fines de estado y de informes.   Los atributos esenciales para el seguimiento son Riesgo, Beneficio, Esfuerzo, Estabilidad e Impacto arquitectónico, para permitir la prioridad de los requisitos para la gestión de ámbitos y para asignar requisitos para las iteraciones.

Correlacionar con herramientas

La rastreabilidad y los atributos son conceptos generales que pueden aplicarse a cualquier producto de trabajo o elemento de producto de trabajo.  Sin embargo, un proyecto típico tendrá herramientas de planificación y de elaboración de presupuestos, herramientas de diseño, herramientas de gestión de requisitos y herramientas de gestión de la configuración.  Estas herramientas a menudo proporcionan y/i imponen determinados atributos y rastreabilidad.

Por ejemplo, las herramientas de planificación suelen proporcionar enlaces entre personas y tareas, y gestionan atributos como puede ser el porcentaje completo.   El enlace de tarea a requisito puede ser implícito mediante convenio de denominación, o puede gestionarse explícitamente.

Las herramientas de diseño proporcionan enlaces entre los elementos de diseño utilizando el Lenguaje unificado de modelado (UML) y gestionan atributos como, por ejemplo, "Descripción", "Permanencia", etc.

Para obtener ayuda sobre cómo enlazar la información entre herramientas, consulte las guías de herramientas  asociadas .

Escribir el plan

El plan de gestión de requisitos describe la entrada necesaria para un plan eficaz. La información proporcionada está prevista para que sirva como directriz. La intención de cada sección debe interpretarse en el contexto de un determinado proyecto/producto. Las directrices detalladas se incluyen en la siguiente Directriz: Plan de gestión de requisitos.
