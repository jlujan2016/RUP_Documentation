Producto de trabajo: Visión
Este artefacto define la vista de los interesados del producto que se va a desarrollar, especificada en términos de necesidades y características clase de los interesados. Contiene una esquematización de los requisitos centrales previstos, para proporcionar una base contractual para los requisitos técnicos más detallados.
Objetivo

Este artefacto proporciona una base de alto nivel, a veces contractual, para los requisitos técnicos más detallados que son visibles para los interesados. Captura la "esencia" de la solución concebida en forma de requisitos de alto nivel y restricciones de diseño que dan al lector una visión general del sistema que se desplegará desde una perspectiva de requisitos de comportamiento. Proporciona información de entrada al proceso de aprobación del proyecto y, por lo tanto, está muy relacionado con el Producto de trabajo: Caso de negocio. Comunica los "qué y porqué" fundamentales para el proyecto y es un indicador contra el que deberían validarse todas las decisiones futuras.

Otro nombre que se utiliza para denominar este artefacto es el Documento de requisitos de producto.

Descripción
Esquematización breve

El documento de visión se escribe desde el punto de vista de los clientes, centrándose en las características fundamentales del sistema y en los niveles de calidad aceptables. La visión debería incluir una descripción de las características que se deben incluir así como de aquellas que se han considerado pero que no se incluyen. También debe especificar las capacidades operativas (volúmenes, tiempos de respuesta, precisiones), los perfiles de usuario (quién utilizará el sistema) y las interfaces interoperativas con entidades fuera de los límites del sistema, cuando sea aplicable.

Descripción principalEl documento de Visión ofrece una visión completa para el sistema de software que se está desarrollando y da soporte al contrato entre la autoridad de financiación y la organización de desarrollo. Cada proyecto necesita una fuente para capturar las expectativas entre los interesados.

Personalización
Opciones de representación

Personalice este artefacto según sea necesario para las necesidades de su proyectos. En general, es una buena práctica que este artefacto sea breve para poder entregarlo a los interesados lo antes posible y para que los interesados lo puedan revisar y absorber fácilmente. Para ello, deben incluirse sólo las solicitudes y características más importantes para los interesados y evitarse los requisitos detallados. Los detalles se pueden incluir en los otros artefactos de requisitos o en apéndices.

Es importante expresar este artefacto en términos de sus guiones de uso y casos de ejemplo principales a medida que se desarrollan, de modo que pueda ver cómo los guiones de uso ejecutan este artefacto.

Decida si los atributos de características (Atributos de requisito) se documentan en la Visión o en el Plan de gestión de requisitos. Decida qué información  se incluirá en la Visión y qué se gestionará con herramientas de gestión de requisitos como Rational RequisitePro (consulte Guía de la herramienta: Desarrollo de una visión utilizando Rational RequisitePro ).
