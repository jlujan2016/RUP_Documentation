Actividad: Preparar el entorno para una iteración
Esta actividad prepara el entorno de desarrollo de un proyecto para la iteración actual, en el que el entorno de desarrollo incluye tanto el proceso como las herramientas.
Amplía: Preparar el entorno para una iteración
DescripciónEstructura de desglose de trabajoAsignación de equiposUtilización del producto de trabajo
Objetivo
La finalidad de esta actividad es garantizar que el entorno del proyecto esté listo para la próxima iteración. Esto incluye el proceso y las herramientas:
Relaciones
Actividades principales
  • Iteración de transición
Descripción

Para cada iteración, esta actividad se centra fundamentalmente en:

  • Completar el guión de desarrollo para prepararse para la iteración.
  • Preparar y, si es necesario, personalizar las herramientas que deben utilizarse para la iteración.
  • Verificar que las herramientas se han configurado e instalado correctamente.
  • Preparar un conjunto de plantillas y directrices específicas del proyecto para dar soporte al desarrollo de artefactos de proyecto en la iteración.
  • Garantizar que todos los cambios efectuados en el entorno del proyecto se comunican adecuadamente a los miembros del proyecto.

Personal

Un ingeniero de proceso es responsable de completar el guión de desarrollo. No es habitual que un individuo pueda dominar todas las disciplinas del desarrollo de software, por lo que es importante que el ingeniero de proceso consulte con expertos, tanto dentro como fuera de la empresa de desarrollo.

El desarrollo de plantillas y directrices específicas del proyecto pueden llevarlo a cabo varias personas. No hay necesidad de un conocimiento detallado del proceso para desarrollar estos artefactos específicos del proyecto.

Suele haber varios individuos que son especialistas en herramientas. Cada individuo es el responsable de una herramienta o de un grupo de herramientas relacionadas.

Utilización
Instrucciones de utilizaciónComo su nombre indica, esta actividad se realiza al principio de cada iteración.
Factores clave

Esta actividad es recomendable para cualquier proyecto, aunque las tareas descritas pueden realizarse de manera un tanto informal en algunos tipos de proyectos y pueden considerarse parte del soporte continuo del entorno del proyecto. Para obtener más información sobre este soporte continuo, consulte la Actividad: Dar soporte al entorno durante una iteración.
