Instrucciones de la herramienta: Adición de elementos al control de origen utilizando Rational ClearCase
En esta guía de la herramienta se describe cómo añadir elementos al control de origen utilizando Rational ClearCase.

Descripción principal

Visión general

En el núcleo de ClearCase se encuentran sus amplias posibilidades para controlar y gestionar versiones de elementos como, por ejemplo, archivos de origen para software y documentación, directorios e informes, entre otros. Antes extraer uno o más elementos, debe aportar la versión inicial de dichos elementos al control de origen.

Pasos de la herramienta

Para añadir uno o más elementos al control de origen utilizando Rational ClearCase, debe llevar a cabo los pasos siguientes:

  • Iniciar el Explorador de ClearCase
  • Navegar a los elementos que se deben transferir al control de origen
  • Seleccionar los elementos
  • Añadir los elementos al control de origen
  • Completar la operación del control de origen

    Más información

Iniciar el Explorador de ClearCase

  1. En el menú Inicio, pulse Programas > Software Rational > Rational ClearCase > Explorador de ClearCase
  2. Seleccione la vista que señala a la VOB a la que desea añadir los elementos para el control de origen.


  3. Si la vista no existe, primero debe crearla. Si la vista existe, pero no ve la VOB, primero debe montar la VOB que va a contener los elementos.

Navegar a los elementos que se deben transferir al control de origen

Se recomienda que seleccione los elementos que se encuentran más alejados del árbol del directorio raíz. Para cualquier elemento dado, ClearCase también añade los directorios padre (hasta el directorio raíz de la VOB) que aún no están en el control de origen.

Nota: Antes de añadir un elemento a una VOB, se debe haber extraído el directorio padre correspondiente. Si aún no lo ha hecho, ClearCase lo extrae antes de crear el elemento y lo incorpora después de crear el elemento.

Seleccionar los elementos

  1. Puede seleccionar uno o más elementos que se deben transferir al control de origen utilizando los métodos de selección estándar.

Añadir los elementos al control de origen

  1. Pulse el botón derecho del ratón sobre los archivos seleccionados y elija Añadir al control de origen en el menú emergente.
  2. Para añadir elementos a un proyecto UCM (Gestión unificada de cambios), en el recuadro de diálogo Añadir al control de origen seleccione o cree una actividad de ClearCase.
  3. En el recuadro Comentario, describa los elementos que está añadiendo al control de origen.
  4. Para continuar trabajando en los nuevos elementos después de añadirlos al control de origen, seleccione la opción Extraer después de añadir al control de origen. Para dejarlos incorporados, deseleccione la opción.
  5. Si trabaja en una VOB replicada (MultiSite) y no utiliza una vista UCM, puede hacer que la réplica actual controle todas las ramificaciones creadas durante la operación de Añadir al control de origen seleccionando la opción Convertir la réplica actual en maestra de todas las ramificaciones recién creadas. Una vez que haya creado el elemento, puede crear nuevas versiones del elemento en cualquiera de las nuevas ramificaciones.

  6. Si no selecciona esta opción, la maestría de la ramificación principal del elemento se asigna a la réplica que controla el tipo de ramificación principal. Si la réplica maestra no es la réplica actual, no se puede crear ninguna versión nueva en la ramificación. Además, si la especificación de la configuración contiene reglas de creación automática de ramificaciones, la creación de las ramificaciones da error si la réplica actual no controla los tipos de ramificaciones.

Completar la operación del control de origen

Complete la operación tal como se indica a continuación:

  1. Para añadir sólo un elemento al control de origen, pulse Aceptar.
  2. Para añadir varios elementos al control de origen, aplicar el mismo comentario y extraer a todos los elementos, pulse Aplicar a todos.
  3. Para cancelar la operación, pulse Cancelar. Los elementos no se transfieren al control de origen y permanecen como archivos de vista privada.
