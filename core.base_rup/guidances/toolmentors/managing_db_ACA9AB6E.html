Comparar y sincronizar

Rational XDE compara un modelo de datos con un esquema de base de datos o archivo de script DDL existente. (Consulte el apartado Comparación del modelo de datos .) A continuación, se pueden sincronizar las diferencias entre el modelo y la base de datos o DDL. (Consulte el apartado Sincronización del modelo de datos .) Puede iniciar el proceso de comparación y sincronización utilizando el Asistente para la sincronización del modelo de datos con la base de datos. (Consulte el apartado Comparación y sincronización del modelo de datos .)

Rational XDE necesita se compare y sincronice el modelo de datos desde el nivel de la base de datos. El esquema de base de datos o DDL que se compara debe tener el mismo nombre que la base de datos del modelo de datos. También debe realizar la comparación con respecto a una base de datos o DDL que utilice el mismo DBMS (sistema de gestión de bases de datos) que la base de datos del modelo de datos.

Consulte el tema de la ayuda en línea de Rational XDE que trata sobre Comparación y sincronización del modelo de datos .

Para el proceso de comparación y sincronización se necesita una conexión con la base de datos. Puede obtener más información sobre el establecimiento de conexiones de base de datos para un DBMS específico en los temas de la ayuda en línea de Rational XDE que tratan sobre:

  • Conexiones de base de datos Oracle
  • Conexiones de base de datos SQL Server
  • Conexiones de base de datos Sybase
  • Conexiones de base de datos DB2
