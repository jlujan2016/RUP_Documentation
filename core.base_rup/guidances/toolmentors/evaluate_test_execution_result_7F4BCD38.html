En esta guía de la herramienta se describe cómo utilizar Rational TestManager, Rational Robot y Rational TestFactory para analizar scripts de prueba generados automáticamente en los que se revelan errores.

Visión general

Esta guía de la herramienta se aplica al ejecutar Windows 98/2000/NT 4.0.

Para utilizar Rational TestFactory, Rational Robot y Rational TestManager a fin de analizar los scripts de prueba generados automáticamente, lleve a cabo una o ambas de las acciones siguientes:

  • Analizar un script de prueba que revela un defecto
  • Analizar un script de prueba que revela una ventana activa inesperada

1. Analizar un script de prueba que revela un defecto

Tal como se ha descrito en la Guía de la herramienta: Implementación de scripts de prueba que se han generado utilizando Rational Test Factory, un piloto es la herramienta de Rational TestFactory que genera el script de prueba automáticamente. Para cada defecto que encuentra mientras se ejecuta, un piloto genera un "script de defecto", que contiene las sentencias del script de prueba que causan que se produzca el defecto.

Cuando un piloto encuentra defectos durante la ejecución, el script de prueba que descubre los defectos se sitúa en una subcarpeta "Defectos encontrados" separada en la carpeta de la ejecución. Puede utilizar Rational TestManager para ver el registro de ejecución correspondiente al script de prueba que ha descubierto defectos.

 Consulte el tema de la ayuda en línea de Rational TestFactory que trata sobre la visualización del registro para una ejecución de script.

2. Analizar un script de prueba que revela una ventana activa inesperada 

Durante la generación del script de prueba, un piloto puede encontrar una "ventana activa inesperada" (UAW), es decir, una ventana que refleja una incoherencia entre los objetos UI de la correlación de aplicaciones y los controles de la aplicación que se somete a prueba (AUT). El piloto puede encontrar una ventana activa inesperada debido a una de las dos condiciones siguientes:

  • La AUT contiene una ventana para la que no existe ningún objeto UI correspondiente en la correlación de aplicaciones. Un ejemplo típico de esta condición es una ventana de mensaje que no se descubre durante la correlación.
  • Un objeto UI de la correlación de aplicaciones representa una ventana que ya no es la AUT.

Cada vez que encuentra una ventana activa inesperada, el piloto genera un "script UAW" y lo coloca en la carpeta de ejecución del piloto. Puede reproducir un script UAW en Rational Robot y, a continuación, utilizar Robot y Rational TestManager para determinar la causa de la ventana activa inesperada. Después de resolver la causa en Rational TestFactory, las ejecuciones subsiguientes del piloto no encuentran la ventana activa inesperada.

  Consulte los temas de la ayuda en línea de Rational TestFactory que tratan sobre:

  • Búsqueda de la causa de una ventana activa inesperada
  • Resolución de una ventana activa inesperada
