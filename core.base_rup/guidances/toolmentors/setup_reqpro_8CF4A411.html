Descripción principal

Visión general

Para crear un nuevo proyecto en RequisitePro, debe utilizar una plantilla de proyecto. Todas las plantillas de proyecto que se entregan con RequisitePro (excepto la plantilla en blanco) incluyen atributos, tipos de requisitos y tipos de documentos predefinidos que utiliza el proyecto que se crea. Puede utilizar la plantilla de caso de uso para crear un proyecto; con esta plantilla, puede utilizar todas las características de RUP. En el nuevo proyecto, puede empezar inmediatamente a crear nuevos documentos de requisitos en Microsoft® Word o requisitos individuales en vistas. Consulte los temas preliminares de la parte superior de la aplicación de ayuda Vamos de RequisitePro para obtener más información sobre cómo empezar el proyecto de RequisitePro.

Si dispone de licencia para un conjunto de aplicaciones de Rational u otros productos de Rational, puede integrar el proyecto de RequisitePro con el trabajo de dichos productos.

  • Para obtener información sobre la integración de RequisitePro con Rational Rose®, consulte la Guía de la herramienta: Gestión de casos de uso con Rational Rose y Rational RequisitePro.
  • Para obtener información sobre la adición de un proyecto de RequisitePro a un proyecto de Rational, consulte la publicación Rational Suite Administrator's Guide del CD de Documentación en línea de Soluciones Rational para Windows. Si desea integrar RequisitePro con Rational ClearQuest®, debe añadir un proyecto de RequisitePro a un proyecto de Rational.
  • Para obtener información sobre el establecimiento de la integración entre RequisitePro y ClearQuest, consulte la publicación Rational Suite Administrator's Guide del CD de Documentación en línea de Soluciones Rational para Windows. Para obtener instrucciones paso a paso para la asociación de registros de ClearQuest con requisitos de RequisitePro, consulte la Guía de la herramienta: Gestión de solicitudes de los interesados utilizando Rational ClearQuest y Rational RequisitePro.

Acerca de la plantilla de proyecto de caso de uso de RequisitePro

La plantilla de caso de uso proporciona los tipos de requisitos de RequisitePro siguientes, que se correlacionan con los elementos de rastreabilidad más importantes tal como se definen en el Concepto: Rastreabilidad. La plantilla también facilita otros tipos de requisitos que se utilizan comúnmente en los proyectos.

Elementos de rastreabilidad de RUP Tipo de requisito de RequisitePro
Necesidades del usuario/interesado (de Visión) STRQ: Tipo de requisito de solicitud del interesado. Una necesidad se identifica marcándola con una prioridad. (La prioridad "Alta" indica una "necesidad").
Característica del producto (de Visión) FEAT: Tipo de requisito de característica
Requisito suplementario (de Especificaciones suplementarias)  SUPL: Tipo de requisito suplementario
 Caso de uso UC: Tipo de requisito de caso de uso
Sección de caso de uso (secciones de un caso de uso detallado) UC: Tipo de requisito de caso de uso -como requisitos hijo de un caso de uso padre (consulte la visión general de relaciones jerárquicas de la ayuda en línea de RequisitePro).
Elemento de diseño (del modelo de diseño) Gestionado en Rose. Consulte la gestión de casos de uso integrada de la ayuda en línea de RequisitePro.

Se ofrece la opción de incluir un tipo de requisito separado para la Sección de caso de uso , en lugar de utilizar requisitos de caso de uso jerárquicos. Resulta útil si se desea que las secciones de caso de uso del proyecto tengan atributos, privilegios de acceso o reglas de rastreabilidad diferentes (por ejemplo, para garantizar que se efectúe el rastreo de casos de uso por secciones de caso de uso , no por casos de uso ).

Pasos de la herramienta

Para establecer RequisitePro para un proyecto, siga los pasos que se indican a continuación:

  1. Crear un proyecto en RequisitePro
  2. Establecer la seguridad del proyecto

1. Crear un proyecto en RequisitePro

  1. Pulse Archivo > Nuevo > Proyecto. Se muestra el recuadro de diálogo Crear proyecto.
  2. Pulse el icono Plantilla de caso de uso y, a continuación, pulse Aceptar. Se muestra el recuadro de diálogo Propiedades de proyecto de RequisitePro.
  3. Escriba un nombre para el proyecto. En el recuadro de texto Directorio se muestra el directorio en el que se van a almacenar los archivos del proyecto. Acepte el directorio por omisión, o bien, pulse el botón Examinar para navegar a otro directorio.
  4. En la lista Base de datos, pulse la base de datos adecuada para utilizar con el proyecto. Si elige Oracle o SQL Server, pulse Propiedades. Complete la información necesaria del recuadro de diálogo Propiedades la base de datos y, a continuación, pulse Aceptar.
  5. Escriba una descripción para el nuevo proyecto y pulse Aceptar para cerrar el recuadro de diálogo.
  6. En la solicitud en la que se le pregunta si desea que RequisitePro cree un nuevo directorio de proyecto, pulse Sí. RequisitePro crea el nuevo proyecto.
  7. Pulse Cerrar para cerrar el recuadro de diálogo Crear proyecto. El nuevo proyecto incluye tipos de documentos, tipos de requisitos y atributos predefinidos. Para revisarlos o modificarlos, seleccione el proyecto en el Explorador, pulse Archivo > Propiedades y, a continuación, pulse las pestañas correspondientes.

2. Establecer la seguridad del proyecto

Puesto que RequisitePro permite que varios usuarios accedan a la misma base de datos y documentos del proyecto simultáneamente, la seguridad del proyecto resulta crucial. Puede habilitar o inhabilitar la seguridad de un proyecto, dependiendo de las necesidades del grupo. Si se habilita la seguridad, los usuarios pertenecen a grupos, y los administradores de RequisitePro asignan permisos específicos a los grupos. Los permisos determinan el acceso o el tipo de acceso de los usuarios a los proyectos.

Todos los usuarios de RequisitePro pueden crear un nuevo proyecto. Cuando crea un nuevo proyecto, se le considera como administrador del proyecto y se le coloca, automáticamente, en el grupo de administradores. Después de crear el proyecto, puede (así como los demás administradores de proyectos) definir otros usuarios y grupos.

Si no se habilita la seguridad, cualquier usuario puede abrir el proyecto de RequisitePro. La primera vez que se abre un proyecto no protegido, RequisitePro le permite iniciar la sesión como el nombre de usuario del campo Inicio de sesión del proyecto por omisión del recuadro de diálogo Opciones (pulse Herramientas > Opciones), como el nombre de inicio de sesión de Windows o como un nombre de usuario que elija. El nombre de usuario se añade a la lista del grupo de administradores del proyecto.

Cualquier administrador del proyecto puede habilitar la seguridad, modificar las listas de Usuarios y Administradores y asignar contraseñas a otros usuarios. Si se habilita la seguridad, el recuadro de diálogo Inicio de sesión del proyecto aparece cada vez que un usuario abre el proyecto.

Para establecer la seguridad del proyecto en RequisitePro:

  1. Pulse Archivo > Administración de proyectos > Seguridad. Se muestra el recuadro de diálogo Seguridad del proyecto.
  2. Seleccione el recuadro de selección Habilitar la seguridad para este proyecto. Se añaden tres grupos por omisión a la lista Grupos: Usuarios suprimidos, Administradores y Usuarios.

Para añadir un grupo de usuarios:

  1. En el recuadro de diálogo Seguridad del proyecto, pulse el botón Añadir, que se encuentra debajo de la lista Grupos. Se muestra el recuadro de diálogo Permisos de grupo.
  2. Escriba un nombre para el grupo.
  3. Seleccione los permisos de proyecto del grupo.
  4. Edite los permisos de tipos de requisitos y documentos.
  5. Pulse Aceptar.

Para añadir un usuario a un grupo:

  1. En la lista Grupos del recuadro de diálogo Seguridad del proyecto, seleccione el grupo que desea que contenga al usuario.
  2. Pulse el botón Añadir, que se encuentra junto a la lista Usuarios de grupo. Se muestra el recuadro de diálogo Añadir usuario.
  3. Escriba un nombre de usuario, una nueva contraseña y la contraseña de verificación.
    Nota: los campos Nueva contraseña y Verificar están cifrados, por lo que muestran un símbolo para cada uno de los 14 caracteres, como máximo. Recuerde: las contraseñas y los nombres de usuario son sensibles a mayúsculas y minúsculas.
  4. Si el usuario tiene una dirección de correo electrónico, escríbala. La dirección de correo electrónico se necesita para participar en discusiones de RequisitePro a través del correo electrónico.
  5. Pulse Aceptar. Se muestra el recuadro de diálogo Seguridad del proyecto. El usuario se añade a la lista del grupo.
  6. Pulse Aceptar.
