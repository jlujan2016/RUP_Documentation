Distribuir el comportamiento de subsistema a elementos de subsistema

Se añaden clases y otros subsistemas al subsistema para realizar las interfaces del subsistema.

Las colaboraciones de elementos de modelo del subsistema se deben documentar utilizando diagramas de secuencia que muestren cómo se realiza el comportamiento del subsistema. Cada operación que se lleva a cabo en una interfaz que realiza el subsistema debe tener uno o más diagramas de secuencia documentados. Este diagrama, que es propiedad del subsistema, se utiliza para diseñar el comportamiento interno del subsistema.

  1. Cree diagramas de clase según sea necesario. Consulte el apartado Adición de diagramas .
  2. Añada subsistemas y clases. Consulte la Guía de la herramienta: Identificación de elementos de diseño utilizando Rational XDE como guía para añadir nuevos subsistemas y clases.
  3. Documente cómo se lleva a cabo cada operación de interfaz, tal como se indica a continuación:
    • Añada una instancia de colaboración para cada operación de interfaz. Denomínela después de la operación. (En el Explorador de modelos, pulse el botón derecho del ratón sobre el paquete subsistema y, a continuación, pulse Añadir UML > Instancia de colaboración).
    • Cree una instancia de interacción con el mismo nombre. (En el Explorador de modelos, pulse el botón derecho del ratón sobre la instancia de colaboración y, a continuación, pulse Añadir UML > Instancia de colaboración).
    • Cree un diagrama de secuencia para la instancia de interacción. Consulte el apartado Adición de diagramas .
    • Arrastre y suelte clases e interfaces en el diagrama para crear objetos para la interacción. Consulte el apartado Adición de formas a diagramas .
    • Añada mensaje o estímulos entre objetos. Consulte el apartado Adición de mensajes o estímulos .
    • Asigne operaciones a los mensajes. Consulte el apartado Asignación de operaciones a mensajes o estímulos .

Para obtener más información, consulte los temas de la ayuda en línea de Rational XDE que tratan sobre:

  • Cómo trabajar con diagramas de clase
  • Cómo trabajar con diagramas de interacción (secuencia)
  • Modelado Java y aplicaciones web

Describir dependencias de subsistema

Cuando un elemento contenido en un subsistema utiliza algo del comportamiento de un elemento contenido en otro subsistema, se crea una dependencia entre los subsistemas incluidos. Para mejorar la reutilización y reducción de las dependencias de mantenimiento, exprese esta situación en términos de una dependencia de una interfaz concreta del subsistema, no en el subsistema en sí mismo ni en el elemento contenido en el subsistema.
