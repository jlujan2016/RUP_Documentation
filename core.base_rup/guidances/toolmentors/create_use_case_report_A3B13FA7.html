Instrucciones de la herramienta: Creación de un informe de guión de uso utilizando Rational SoDA
En esta guía de la herramienta se describe cómo utilizar SoDA para crear un informe de guión de uso.

Descripción principal

Visión general

SoDA automatiza la generación del informe, de forma que se pueda crear rápidamente y con precisión. Puede generar un informe de guión de uso con la versión de Microsoft® Word® o Adobe® FrameMaker® de SoDA.

Información relacionada con Rational Unified Process:

  • Informe: Especificación de guión de uso
  • Producto de trabajo: Guión de uso

Esta guía de la herramienta se aplica al ejecutar Windows 2000, NT 4.0, Windows XP, Solaris o HP-UX.

Para crear un informe de guión de uso utilizando SoDA, siga el procedimiento que se indica para su versión del producto.

  • Utilizar Rational SoDA/Word para generar un informe de guión de uso
  • Utilizar Rational SoDA/FrameMaker para generar un informe de guión de uso

Utilizar Rational SoDA/Word para generar un informe de guión de uso

  1. En un diagrama de guión de uso en Rational Rose, seleccione el guión de uso que desea documentar.
  2. Pulse Informe > Informe SoDA.
  3. Cuando aparezca la lista de informes disponibles en SoDA, seleccione Informe de guión de uso de Rational Unified Process.
    Si este informe no está en la lista, cancele y asegúrese de que ha seleccionado un diagrama de guión de uso.
  4. Pulse Aceptar.
  5. Cuando se le solicite la UseCaseSpecification, entre o busque el archivo de Word que se debe incluir en el informe.
  6. Pulse Aceptar para generar el informe.

2. Utilizar Rational SoDA/FrameMaker para generar un informe de guión de uso

  1. En la barra de botones de FrameMaker, pulse Nuevo. Efectúe una doble pulsación en SoDA y, a continuación, efectúe una doble pulsación en RoseDomain y elija la plantilla RUPUseCaseReport.fm.
  2. Edite el primer conector y entre el nombre del modelo, el paquete y el guión de uso.
  3. Pulse Archivo > Guardar como para guardar la plantilla en un directorio personal o en el directorio del proyecto. Es posible que desee cambiar el nombre de la plantilla de modo que refleje el nombre del guión de uso; por ejemplo, MaintainLoansReport.fm.
  4. Pulse SoDA > Generar documento.
  5. Cuando se le solicite la UseCaseSpecification, entre o busque el archivo de FrameMaker que se debe incluir en el informe.
  6. Pulse Aceptar para generar el informe.
  7. Revise el documento generado.

La próxima vez que desee generar este documento, abra el documento y pulse SoDA > Generar documento.
