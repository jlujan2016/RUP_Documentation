Instrucciones de la herramienta: Ejecución de un conjunto de aplicaciones de prueba utilizando Rational TestFactory
En esta guía de la herramienta se describe cómo utilizar Rational TestFactory para crear y ejecutar un conjunto de aplicaciones de prueba.
Herramienta: Rational TestFactory

Descripción principal

Visión general

Un "conjunto de aplicaciones de prueba" es el elemento de Rational TestFactory que le permite organizar y ejecutar un conjunto de aplicaciones de scripts de prueba. Gracias a este elemento de TestFactory, puede organizar scripts de prueba, ejecutarlos en un proceso por lotes como un conjunto de aplicaciones y. a continuación, analizar la cobertura de la prueba basada en código.

Esta guía de la herramienta se aplica al ejecutar Windows 98/2000/NT 4.0.

Para utilizar Rational TestFactory para ejecutar un conjunto de aplicaciones de scripts de prueba debe:

  1. Crear un conjunto de aplicaciones de scripts de prueba
  2. Ejecutar el conjunto de aplicaciones de prueba

1.  Crear un conjunto de aplicaciones de scripts de prueba

Un objeto de conjunto de aplicaciones de prueba es un contenedor al que se añaden los scripts de prueba que se desea ejecutar como un proceso por lotes. El conjunto de aplicaciones de prueba puede incluir scripts de prueba registrados en Rational Robot, así como scripts de prueba generados en Rational TestFactory.

  • Para crear scripts de prueba en Robot, consulte la Guía de la herramientas: Implementación de scripts de prueba utilizando Rational Robot.
  • Para generar scripts de prueba en TestFactory, consulte la Guía de la herramienta: Implementación de los scripts de prueba que se han generado utilizando Rational TestFactory.

Los conjuntos de aplicaciones de prueba se utilizan para organizar y ejecutar grupos de los mejores scripts, los scripts de defectos y los scripts generados con respecto a una compilación específica de la aplicación que se somete a prueba (AUT). Los scripts de prueba que se incluyen en el conjunto de aplicaciones de prueba permanecen enlazados al objeto del conjunto de aplicaciones de prueba. Puede ejecutar el conjunto de aplicaciones de prueba como un conjunto de aplicaciones de prueba de regresión en futuras compilaciones de la aplicación que se somete a prueba.

Rational TestFactory ofrece dos métodos para crear un conjunto de aplicaciones de prueba:

  • Dejar que TestFactory cree el conjunto de aplicaciones de prueba automáticamente con scripts de prueba seleccionados.
  • Añadir manualmente un objeto de conjunto de aplicaciones de prueba a la correlación de aplicaciones y, a continuación, rellenar el conjunto de aplicaciones de prueba con scripts de prueba.

Si desea analizar la cobertura de la prueba basada en código para los scripts de prueba del conjunto de aplicaciones de prueba, debe instrumentar el AUT antes de ejecutar el conjunto de aplicaciones de prueba. Rational TestFactory sólo calcula la cobertura de la prueba basada en código para los scripts de prueba si se ha instrumentado la aplicación.

  Consulte los temas de la ayuda en línea de Rational TestFactory que tratan sobre:

  • Conjuntos de aplicaciones de prueba: Qué son y cómo funcionan
  • Cómo trabajar con conjuntos de aplicaciones de prueba
  • Visualización de la cobertura de código para un script de Robot
  • Instrumentación de la aplicación que se somete a prueba

2.  Ejecutar el conjunto de aplicaciones de prueba

Cuando ejecuta un conjunto de aplicaciones de prueba, Rational TestFactory ejecuta cada script de prueba en el orden en el que aparece en la lista del conjunto de aplicaciones de prueba. Si el orden de los scripts de prueba es importante, puede reordenarlos antes de ejecutar el conjunto de aplicaciones de prueba.

 Consulte el tema de la ayuda en línea de Rational TestFactory que trata sobre la ejecución manual de un conjunto de aplicaciones de prueba.
