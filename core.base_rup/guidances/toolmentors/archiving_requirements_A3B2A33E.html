Instrucciones de la herramienta: Archivado de requisitos utilizando Rational RequisitePro
En esta guía de la herramienta se describe cómo archivar requisitos utilizando mandatos de archivado de Rational RequisitePro.
Herramienta: Rational RequisitePro
Relaciones

Descripción principal

Visión general

Con fines de copia de seguridad, debe capturar requisitos del proyecto con objetivos distintos. Gracias a RequisitePro, puede tomar una instantánea de los requisitos del proyecto y almacenarla en Rational ClearCase® o, simplemente, como un archivador de RequisitePro, que coloca una copia de los productos de trabajo del proyecto en el directorio del archivador.

Puede utilizar varias herramientas de gestión de la configuración como, por ejemplo, Intersolv PVCS® o Microsoft® Visual SourceSafe, para archivar los proyectos de RequisitePro. Si el proyecto utiliza una base de datos de Microsoft® Access, primero utilice el mandato Archivador de RequisitePro para crear un archivo archivador y, a continuación, archive el archivo utilizando la herramienta de gestión de la configuración. Asegúrese de que incluye todos los documentos del proyecto en el archivador. Cuando archive un proyecto que utilice rastreabilidad entre proyectos, asegúrese de que archiva todos los proyectos asociados a la vez, a fin de que el proyecto archivado esté completo.

Un proyecto de RequisitePro consta de un proyecto de requisitos de Rational y un conjunto de documentos que contienen requisitos. Los requisitos se pueden crear directamente en una vista del proyecto o en un documento de Microsoft® Word en RequisitePro. En el proyecto de RequisitePro asociado que mantiene los requisitos se hace referencia a todos los documentos que contienen requisitos. Un archivador de proyectos de RequisitePro contiene el proyecto de requisitos de Rational y todos los documentos que contienen requisitos.

Nota: para obtener información sobre la creación de una línea base de un proyecto de RequisitePro, consulte el apartado Guía de la herramienta: Creación de una línea base de un proyecto de Rational RequisitePro.

Pasos de la herramienta

Para archivar un proyecto utilizando RequisitePro, elija una de las opciones siguientes:

  1. Archivar un proyecto de RequisitePro como un archivador de RequisitePro
  2. Archivar un proyecto de RequisitePro en ClearCase
  3. Archivar un proyecto de RequisitePro utilizando una base de datos empresarial

1. Archivar un proyecto de RequisitePro como un archivador de RequisitePro

Cuando archiva un proyecto de RequisitePro como un archivador de RequisitePro, simplemente hace una copia de todos los archivos del proyecto en un directorio definido por el usuario. Puede utilizar esta opción para archivar un proyecto de RequisitePro utilizando sistemas de gestión de la configuración para los que RequisitePro no ofrece soporte directamente.

Para archivar un proyecto de RequisitePro como un archivador de RequisitePro, siga los pasos que se indican a continuación:

  1. Pulse Archivo > Administración de proyectos > Archivador > Archivador de RequisitePro. Se muestra el recuadro de diálogo Archivar proyecto.
  2. Seleccione el directorio en el que desea almacenar la copia de seguridad. Pulse Examinar para abrir el recuadro de diálogo Examinar para buscar carpeta. Navegue a un directorio, seleccione una carpeta y pulse Aceptar para volver al recuadro de diálogo Archivar proyecto.
  3. Para añadir un nuevo subdirectorio a una vía de acceso de directorio existente, pulse al final de un campo Directorio y teclee una barra inclinada invertida (\) y el nombre del nuevo subdirectorio.
  4. Para aplicar un número de revisión a los documentos de requisitos, seleccione el recuadro de selección Propagar a todos los documentos.
    Nota: si se ha habilitado la seguridad, debe tener permiso de estructura de proyectos para propagar el número de revisión. Y también, el proyecto se debe abrir con carácter exclusivo.
  5. Pasos opcionales: Teclee un nuevo número de revisión para el proyecto y escriba un nombre para la versión archivada en el recuadro Etiqueta de versión y una descripción para la misma en el recuadro Descripción de cambio.
  6. Pulse Aceptar.

Para acceder a este archivador más tarde, simplemente, añada el proyecto archivado a la lista de proyectos de RequisitePro:

  1. Pulse Archivo > Abrir proyecto. Se muestra el recuadro de diálogo Abrir proyecto.
  2. Pulse Añadir. Busque y seleccione el proyecto archivado en el directorio de archivador especificado.
  3. Pulse Abrir y Aceptar. El proyecto archivado está disponible en RequisitePro.

Para obtener más información

Consulte el tema de la ayuda en línea de RequisitePro que trata sobre el archivado de proyectos con el mandato Archivador de RequisitePro (Índice: archivado>utilización del mandato Archivador).

2. Archivar un proyecto de RequisitePro en ClearCase

Para archivar un proyecto de Rational RequisitePro en Rational ClearCase, siga los pasos que se indican a continuación:

  1. Pulse Herramientas > Opciones.
  2. En el recuadro Vista de ClearCase, teclee la ubicación VOB (base de objetos con versión) de destino (unidad, nombre de vista, nombre de vob y directorio). La vía de acceso debe empezar por una letra de unidad. Utilice nombres del convenio de denominación universal para una unidad compartida. Este paso define la vista en la que se van a copiar los documentos y la base de datos de RequisitePro cuando se archive el proyecto.
  3. Pulse Aceptar para guardar la entrada y cerrar el recuadro de diálogo Opciones.
  4. Pulse Archivo > Administración de proyectos > Archivador > Rational ClearCase. Se muestra el recuadro de diálogo Archivar con ClearCase.
  5. Seleccione el directorio en el que desea almacenar la copia de seguridad. Pulse Examinar para abrir el recuadro de diálogo Examinar para buscar carpeta. Navegue a un directorio, seleccione una carpeta y pulse Aceptar. Vuelve al recuadro de diálogo Archivar con ClearCase.
  6. Para añadir un nuevo subdirectorio a una vía de acceso de directorio existente, pulse al final de un campo Directorio y teclee una barra inclinada invertida (\) y el nombre del nuevo subdirectorio.
  7. Los pasos siguientes son opcionales:
    • Teclee un nuevo número de revisión para el proyecto.
    • Aplique el número de revisión a los documentos de requisitos seleccionando el recuadro de selección Propagar a todos los documentos. Nota: si se ha habilitado la seguridad, debe tener permiso de estructura de proyectos para propagar el número de revisión. Y también, el proyecto se debe abrir con carácter exclusivo.
    • Escriba un nombre para la versión archivada en el recuadro de texto Etiqueta de versión y una descripción para la misma en Descripción de cambio.

Para obtener más información

Consulte el tema de la ayuda en línea de RequisitePro que trata sobre el archivado de proyectos con Rational ClearCase (Índice: archivado>utilizando ClearCase).

3. Archivar un proyecto de RequisitePro utilizando una base de datos empresarial

Para obtener directrices completas sobre el archivado de proyectos de RequisitePro que utilicen una base de datos Oracle o Microsoft SQL Server, consulte el tema de la ayuda en línea de RequisitePro que trata sobre el archivado de un proyecto de base de datos empresarial (Índice: archivado>bases de datos empresariales).
