Instrucciones de la herramienta: Ingeniería directa de bases de datos utilizando Rational XDE
En esta guía de la herramienta se describe el uso de la ingeniería directa en la herramienta de software de Rational XDE Developer para generar un archivo de script DDL o esquema de base de datos a partir de un modelo de datos de Rational XDE existente.

Descripción principal

Visión general

En esta guía de la herramienta se llevan a cabo los pasos siguientes:

  • Establecer propiedades de modelo
  • Ingeniería directa de modelo de datos para una base de datos
  • Gestionar cambios en el modelo y la base de datos

Establecer propiedades de modelo

Se deben establecer dos tipos de propiedades de modelo para permitir la ingeniería directa del modelo de datos para crear una base de datos o archivo DDL (lenguaje de definición de datos). Las propiedades son las siguientes:

  • DBMS (sistema de gestión de bases de datos) por omisión (consulte el apartado Establecimiento del destino de base de datos por omisión )
  • Asignación de base de datos por omisión (consulte el apartado Establecimiento de la asignación de base de datos por omisión )

Rational XDE ofrece soporte de ingeniería directa para el estándar ANSI SQL 92 y para los DBMS (sistemas de gestión de base de datos) listados en el tema de la ayuda en línea de Rational XDE que trata sobre las bases de datos soportadas en Rational XDE .

Referencia: Establecimiento de opciones de modelado de datos .

Ingeniería directa de modelo de datos para una base de datos 

Rational XDE se puede utilizar para aplicar la ingeniería de un modelo de datos a un nuevo esquema de base de datos o un nuevo archivo DDL. Puede aplicar la ingeniería a tablas individuales o a un modelo de datos completo. Referencia: Generación de base de datos y DDL El proceso de ingeniería directa se inicia utilizando el asistente para ingeniería directa de modelado de datos que proporciona Rational XDE. (Consulte el apartado Generación de bases de datos o DDL .)  

Tanto si decide generar sólo un DDL, como si decide ejecutar el DDL para crear una base de datos, Rational XDE crea, automáticamente, un archivo DDL en el directorio que se especifica en el asistente. Los elementos que incluye Rational XDE en el proceso de ingeniería directa dependen de las opciones que se seleccionan en el Asistente para ingeniería directa. Todos los elementos o scripts DDL se generan conforme al DBMS especificado. Consulte los temas de la ayuda en línea de Rational XDE que tratan sobre los detalles de los distintos tipos de DBMS para los que se ofrece soporte:

  • Base de datos DB2 y generación de DDL
  • Base de datos Oracle y generación de DDL
  • Base de datos SQL Server y generación de DDL
  • Base de datos Sybase y generación de DDL

Si aplica la ingeniería a un modelo de datos ANSI SQL 92, Rational XDE genera un DDL independiente de DBMS. Puede utilizar la herramienta de base de datos que prefiera para ejecutar el DDL.

Con el objeto de realizar la ingeniería directa de la base de datos, Rational XDE debe establecer una conexión para los DBMS adecuados. Consulte el apartado Conexión a sistemas de gestión de base de datos  para obtener más información sobre el establecimiento de conexiones de base de datos. Consulte los temas de la ayuda en línea de Rational XDE que tratan sobre el establecimiento de conexiones para un tipo de DBMS específico:

  • Conexiones de base de datos Oracle
  • Conexiones de base de datos SQL Server
  • Conexiones de base de datos Sybase
  • Conexiones de base de datos DB2

Gestionar cambios en el modelo y la base de datos

Después de crear la base de datos o DDL, se deben gestionar y controlar los cambios del modelo de datos y la base de datos generada. Rational XDE proporciona una característica de Comparar y sincronizar para ayudar al diseñador de la base de datos a gestionar el cambio. Referencia: Comparación y sincronización en el modelado de datos , Guía de la herramienta: Gestión de bases de datos en Rational XDE.
