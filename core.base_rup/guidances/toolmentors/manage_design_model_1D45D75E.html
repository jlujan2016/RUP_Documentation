Descripción principal

Visión general

Para gestionar el Producto de trabajo: Modelo de diseño, se deben llevar a cabo los pasos se indican a continuación:

  1. Crear el modelo de diseño
  2. Crear paquetes en el modelo de diseño
  3. Crear capas en el modelo de diseño
  4. Documentar la organización del modelo

1. Crear el modelo de diseño

El Producto de trabajo: Modelo de diseño se puede representar en Rational Rose utilizando un paquete en la vista lógica denominado "Modelo de diseño". El modelo de diseño representa un nivel de abstracción próximo al código fuente. Por ejemplo, en el modelo de diseño las clases tienen, generalmente, unas propiedades de modelo y lenguaje asignadas que definen la correlación con código.

2. Crear paquetes en el modelo de diseño

Un paquete es un elemento de modelo de finalidad general que organiza elementos de modelo en grupos. Se puede considerar que un sistema es un único paquete de alto nivel, con todo lo demás del sistema contenido en éste.

Para obtener más información sobre los paquetes, consulte el tema de la ayuda en línea de Rational Rose que trata sobre los paquetes.

3. Crear capas en el modelo de diseño

Un modelo puede contener varias capas, que se visualizan como paquetes, un paquete por capa. Las capas se pueden anidar. Para distinguir paquetes como capas, debe añadir el estereotipo "capa" a cada paquete. Además, debe utilizar el campo Documentación de la especificación de paquete para describir el paquete.

4. Documentar la organización del modelo

Todos los modelos necesitan un diagrama de clase en el que se muestre la organización de capas y subsistemas del modelo. Debe ser el diagrama Principal. Considere renombrar el diagrama "Organización del modelo". Arrastre y suelte los paquetes que representan las capas o los subsistemas al diagrama.
