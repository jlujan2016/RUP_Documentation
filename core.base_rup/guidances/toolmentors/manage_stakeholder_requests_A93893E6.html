Instrucciones de la herramienta: Gestión de solicitudes de los interesados utilizando Rational ClearQuest y Rational RequisitePro
En esta guía de la herramienta se describe cómo crear y asociar requisitos en Rational RequisitePro® con solicitudes de mejora en Rational ClearQuest®, lo que permite que el analista pueda hacer un seguimiento eficaz del origen de los requisitos. También se describe cómo gestionar propiedades de requisitos desde ClearQuest.
Herramienta: Rational Suite AnalystStudio

Descripción principal

Nota: Para acceder a esta funcionalidad, se debe haber instalado tanto RequisitePro como ClearQuest, y se debe haber configurado un proyecto del administrador de Rational con ambos productos.

Visión general

El rol del analista consiste en incorporar la información de retorno del usuario y las necesidades de los interesados en la definición del sistema. Gracias a la integración de Rational entre ClearQuest y RequisitePro, se pueden asociar fácilmente solicitudes de mejora calificadas con requisitos relacionados, permitiendo así el seguimiento del origen de los requisitos.

Para que cada solicitud de mejora identificada dirija la funcionalidad del sistema desde ClearQuest, el analista puede crear o asociar uno o varios requisitos formulando de nuevo el texto de la solicitud de mejora en una definición de requisitos válida. Una definición de requisitos bien definida hace referencia a los requisitos que se pueden probar, sin ambigüedades, coherentes, verificables y completos. Los requisitos se almacenan en RequisitePro, pero se pueden crear en ClearQuest o en RequisitePro, y sus propiedades se pueden gestionar tanto desde ClearQuest como desde RequisitePro.

Nota: Sólo puede asociar solicitudes de mejora con requisitos en aplicaciones Windows de RequisitePro o ClearQuest, no en RequisiteWeb ni el la web de ClearQuest.

Pasos de la herramienta

Este documento contiene los pasos siguientes:

  1. Habilitar la integración de ClearQuest y RequisitePro
  2. Utilizar ClearQuest para seleccionar un proyecto de RequisitePro
  3. Utilizar ClearQuest para asociar requisitos con solicitudes de mejora
  4. Utilizar ClearQuest para crear requisitos de solicitudes de mejora
  5. Gestionar propiedades de requisitos desde ClearQuest

1. Habilitar la integración de ClearQuest y RequisitePro

Para asociar solicitudes de mejora con requisitos se necesita una configuración mínima. Dicha configuración implica la utilización de un proyecto de Rational Suite para señalar a la base de datos de ClearQuest y al proyecto de RequisitePro. En la configuración, el administrador de ClearQuest define los tipos de requisitos que se pueden crear desde ClearQuest. Los administradores definen una correlación a fin de automatizar las necesidades específicas del proyecto. Para obtener información sobre la habilitación de la integración, consulte el documento titulado Rational Suite Administrator's Guide del CD de documentación.

2. Utilizar ClearQuest para seleccionar un proyecto de RequisitePro 

Para asociar solicitudes de mejora con requisitos, cada solicitud de mejora debe identificar un proyecto de Rational Suite que se utilice como origen de los requisitos disponibles.

Para seleccionar un proyecto de RequisitePro desde ClearQuest:

  1. Seleccione una solicitud de mejora y pulse Acciones > Modificar para editar el registro.
  2. Pulse la pestaña Principal.
  3. La lista Proyecto RA muestra los proyectos de Rational Suite válidos. Cada proyecto de Rational Suite se asocia con un proyecto de RequisitePro exclusivo. Elija el proyecto adecuado y pulse Aplicar. ClearQuest guarda la referencia del proyecto con el registro de la solicitud de mejora.

3. Utilizar ClearQuest para asociar requisitos con solicitudes de mejora

Después de identificar un proyecto de Rational Suite en ClearQuest, el analista puede asociar solicitudes de mejora con requisitos. Esta asociación sólo se puede llevar a cabo una vez que se ha enviado la solicitud de mejora, no en el momento del envío. Si un requisito existente representa las necesidades de los interesados expresadas en la descripción de la solicitud de mejora, el analista asocia la solicitud de mejora con el requisito existente seleccionando el requisito de la lista completa de requisitos. Se pueden asociar varias solicitudes de mejora con un requisito, y se pueden asociar varios requisitos con una solicitud de mejora.

Para asociar una solicitud de mejora con un requisito existente:

  1. Seleccione una solicitud de mejora y pulse Acciones > Modificar para editar el registro. Puede seleccionar varias solicitudes de mejora para asociarlas con el mismo requisito pulsando la tecla Control o DESPL mientras selecciona las solicitudes de mejora.
  2. En la pestaña Principal, seleccione un Proyecto RA.
  3. Pulse la pestaña Requisitos. Esta pestaña se utiliza para seleccionar o eliminar requisitos asociados, así como visualizar las propiedades de los requisitos asociados.
  4. Pulse el botón Añadir a lista. Se muestra el recuadro de diálogo Seleccionar requisito. Los requisitos que se muestran en este recuadro de diálogo se consultan de modo dinámico desde el proyecto de RequisitePro asociado.
  5. En la lista Requisitos de tipo, seleccione el tipo de requisito FEAT. Se muestran todos los requisitos FEAT.
  6. Seleccione el requisito que desea y pulse Aceptar.
  7. Pulse Aplicar para guardar los cambios.

Para obtener información detallada, consulte la Guía de la herramienta: Deducción de solicitudes de los interesados utilizando Rational ClearQuest.

4. Utilizar ClearQuest para crear requisitos de solicitudes de mejora

Si ningún requisito representa la necesidad del interesado que se refleja en la definición de la solicitud de mejora, el analista puede crear un nuevo requisito desde ClearQuest.

  1. Seleccione una solicitud de mejora y pulse Acciones > Modificar para editar el registro. Puede seleccionar varias solicitudes de mejora para asociarlas con el mismo requisito pulsando la tecla Control o DESPL mientras selecciona las solicitudes de mejora.
  2. En la pestaña Principal, seleccione un Proyecto RA.
  3. Pulse la pestaña Requisitos. Esta pestaña se utiliza para seleccionar o eliminar requisitos asociados, así como visualizar las propiedades de los requisitos asociados.
  4. Pulse el botón Añadir a lista. Se muestra el recuadro de diálogo Seleccionar requisito. Los requisitos que se muestran en este recuadro de diálogo se consultan de modo dinámico desde el proyecto de RequisitePro asociado.
  5. Seleccione un tipo de requisito para el nuevo requisito.
  6. Pulse Crear. Se muestra el recuadro de diálogo Propiedades del requisito.
  7. Pulse la pestaña General y escriba el nombre y el texto del requisito. Asegúrese de que selecciona el paquete adecuado para que contenga el nuevo requisito.
  8. Pulse la pestaña Atributos y establezca los valores del atributo.
  9. Pulse Aceptar. Se vuelve a mostrar el recuadro de diálogo Seleccionar requisito.
  10. Desplácese a la parte inferior de la lista de requisitos para localizar el requisito que acaba de crear, resáltelo y pulse Aceptar.
  11. Pulse Aplicar para guardar los cambios. El requisito se asocia a la solicitud de mejora.

Nota: Los requisitos que se crean de este modo se ubican sólo en la base de datos, no en documentos. En RequisitePro, puede reubicar el requisito en un documento para su posterior elaboración cortando el requisito de una vista (por medio de Editar > Cortar) y pegándolo en el documento de RequisitePro que desea (por medio de RequisitePro > Requisito > Pegar).

Para obtener más información sobre la elaboración de requisitos de características en un documento, consulte la Guía de la herramienta: Desarrollo de una visión utilizando Rational RequisitePro.

5. Gestionar propiedades de requisitos desde ClearQuest

Desde ClearQuest se puede acceder directamente a las propiedades de requisitos para los requisitos creados en una base de datos. Después de asociar la solicitud de mejora con un requisito, el analista puede revisar y modificar las propiedades del requisito del requisito asociado mientras trabaja directamente en ClearQuest. El recuadro de diálogo Propiedades del requisito facilita la visualización y edición de propiedades de requisitos. Las propiedades incluyen atributos de requisitos, historial y relaciones.

  1. Seleccione una solicitud de mejora y pulse la pestaña Requisitos. Esta pestaña se utiliza para ver las propiedades del requisito asociado.
  2. Pulse Acciones > Modificar para editar el registro.
  3. Seleccione el requisito asociado y pulse Propiedades, que se encuentra en la parte inferior del formulario. Se muestran las propiedades del requisito.
  4. Realice los cambios en el requisito.
  5. Pulse Aceptar para guardar los cambios.

Consulte el tema de la ayuda en línea de RequisitePro que trata sobre visión general de las propiedades del requisito (Índice: requisitos > propiedades).
