Instrucciones de la herramienta: Establecimiento de políticas utilizando Rational ClearCase
En esta guía de la herramienta se describe cómo establecer políticas de proyecto con UCM (Gestión unificada de cambios) de Rational ClearCase.
Herramienta: Rational ClearCase
Relaciones

Descripción principal

Visión general

Las políticas de proyecto permiten la imposición de recomendaciones de desarrollo en un equipo. Al establecer políticas puede, por ejemplo, minimizar los problemas que se pueden producir al integrar trabajo estableciendo una política que requiera que los desarrolladores actualicen sus áreas de trabajo con líneas base recomendadas del proyecto antes de entregar trabajo a la corriente de datos compartida. Esta práctica reduce la posibilidad de incluir defectos de código.

Las políticas de proyecto se establecen al crear un proyecto de UCM, y se pueden modificar en cualquier momento.

Esta guía de la herramienta se aplica al ejecutar Microsoft Windows.

Pasos de la herramienta

Para establecer políticas para proyectos de UCM existen dos procedimientos:

  • Establecer políticas para nuevos proyectos
  • Establecer o cambiar políticas para proyectos existentes

1. Establecer políticas para nuevos proyectos

  • Desde la barra de tareas de Windows, seleccione Inicio > Programas > Software Rational > Rational ClearCase > Explorador de proyectos.
  • En el menú Archivo, seleccione Nuevo > Proyecto para iniciar el asistente Proyecto nuevo. Cuando se le solicite, seleccione las políticas que imponer para este proyecto.

Para obtener detalles, consulte el tema de la ayuda en línea que trata sobre la gestión de proyectos de software.

2. Establecer o cambiar políticas para proyectos existentes

  1. En el Explorador de proyectos, localice el proyecto que desea modificar.
  2. Pulse el botón derecho del ratón sobre el proyecto y seleccione Políticas.
  3. Pulse las diferentes pestañas para visualizar y seleccionar políticas de proyecto.

Consulte el tema de la ayuda según contexto para el diálogo Políticas para obtener una descripción de cada política de proyecto.

Consulte el capítulo que trata sobre el "establecimiento de políticas" del manual IBM Rational ClearCase Guide to Managing Software Projects para obtener más información sobre las políticas que se pueden establecer para proyectos de UCM.
