Instrucciones de la herramienta: Creación de conjuntos de aplicaciones de pruebas de rendimiento con Rational TestManager
En esta guía de la herramienta se describe cómo utilizar Rational TestManager para crear un conjunto de aplicaciones de prueba de rendimiento.
Herramienta: Rational TestManager
Relaciones
Elementos relacionados
  • Estructurar la implementación de la prueba
  • Implementar el conjunto de aplicaciones de prueba
Descripción principal

Visión general

Un conjunto de aplicaciones muestra una representación jerárquica de la carga de trabajo y la población de usuarios que se emulan en una prueba de rendimiento. Muestra elementos como, por ejemplo, grupos de usuarios, los scripts que ejecuta cada grupo y el número o porcentaje de verificadores virtuales asignado a cada grupo.

Esta guía de la herramienta se aplica a plataformas Microsoft® Windows 98/2000/NT 4.0.

Pasos de la herramienta

Para diseñar conjuntos de aplicaciones de prueba de rendimiento automatizadas utilizando TestManager, efectúe los pasos siguientes:

  1. Crear un conjunto de aplicaciones
  2. Insertar grupos de usuarios en un conjunto de aplicaciones
  3. Insertar scripts de prueba en un conjunto de aplicaciones

1. Crear un conjunto de aplicaciones

Un conjunto de aplicaciones le permite ejecutar scripts de prueba y, lo que es más importante, emular las acciones de usuarios reales que acceden a la aplicación multiusuario. Un conjunto de aplicaciones puede ser tan simple como un verificador virtual que ejecuta un script de prueba, o tan complejo como millares de verificadores virtuales que ejecutan diversos scripts de prueba.

Puede crear un conjunto de aplicaciones de rendimiento utilizando uno de los procedimientos siguientes:

  • Utilizando el asistente para el conjunto de aplicaciones de prueba de rendimiento
  • Basándose en una sesión de Robot existente
  • Utilizando un conjunto de aplicaciones de prueba de rendimiento en blanco

En esta guía de la herramienta se describe la utilización de un conjunto de aplicaciones de prueba de rendimiento en blanco.

Para crear el conjunto de aplicaciones:

  1. Pulse Archivo > Nuevo conjunto de aplicaciones.
  2. Pulse Conjunto de aplicaciones de prueba de rendimiento en blanco.

   Consulte el tema de la ayuda en línea de TestManager que trata sobre la creación de un nuevo conjunto de aplicaciones.

2. Insertar grupos de usuarios en un conjunto de aplicaciones

Los grupos de usuarios son el bloque de compilación básico para todos los conjuntos de aplicaciones de prueba de rendimiento. Un grupo de usuarios es una recopilación de verificadores virtuales que llevan a cabo el mismo conjunto de actividades. Todos los conjuntos de aplicaciones de prueba de rendimiento deben contener uno o más grupos de usuarios, fijos o escalables.

Por lo general, los grupos de usuarios escalables permiten más flexibilidad que los grupos de usuarios fijos. Con los grupos de usuarios escalables, un solo conjunto de aplicaciones de prueba de rendimiento puede emular un número diferente de usuarios en cada ejecución de prueba. Si el conjunto de aplicaciones de prueba contiene grupos fijos, y desea ejecutar pruebas con números diferentes de verificadores virtuales, debe modificar el número de verificadores virtuales de cada grupo de usuarios del conjunto de aplicaciones en el tiempo de ejecución de la prueba.

Para insertar un grupo de usuarios en un conjunto de aplicaciones abierto:

  1. Pulse Conjunto de aplicaciones > Insertar > Grupo de usuarios.
  2. Elija entre grupos de usuarios Fijos o Escalables.
  3. Establezca el Número de usuarios (para un grupo de usuarios fijo) o el Porcentaje (para un grupo de usuarios escalable). Si el porcentaje para un grupo de usuarios escalable es inferior al 100 %, se debe añadir otro grupo de usuarios (el porcentaje de la suma total de todos los grupos de usuarios debe ser 100).

3.   Insertar scripts de prueba en un conjunto de aplicaciones

En un conjunto de aplicaciones abierto, efectúe los pasos siguientes:

  1. Seleccione los grupos de usuarios que van a ejecutar el script de prueba.
  2. Pulse Conjunto de aplicaciones > Insertar > Script de prueba.
  3. Cuando aparezca el diálogo Ejecutar propiedades del script de prueba, se pueden establecer las propiedades siguientes:

  • Origen de script de prueba - Elija el tipo de script (GUI, VU, VB, Java) que desea insertar.

  • Consulta - Especifique las características de los scripts que desea insertar en el conjunto de aplicaciones. Si el almacén de datos contiene un gran número de scripts de prueba, compile una consulta para que sólo se muestren los scripts que satisfagan ciertos criterios (por ejemplo, creados por una persona determinada o modificados a partir de una fecha concreta).

  • Condición previa - Selecciónelo para especificar si la terminación satisfactoria de un script de prueba es una condición previa para el resto de la secuencia del conjunto de aplicaciones. Significa que el script de prueba se debe haber completado satisfactoriamente para que se ejecuten elementos subordinados de la secuencia del conjunto de aplicaciones.

  • Iteraciones - Especifique la frecuencia con la que se deben repetir los scripts seleccionados.

  • Método de planificación - Especifique el retardo, si procede, antes del inicio del script, o bien, especifique otros sucesos que desencadenen la ejecución del script.

  1. Una vez que haya establecido las propiedades que desea, seleccione el nombre de la prueba que insertar y pulse Aceptar.

Nota: No puede mezclar scripts de prueba de GUI y VU en un grupo de usuarios. Sin embargo, puede mezclar otros tipos de script de prueba.
