Instrucciones de la herramienta: Creación de un documento de arquitectura de software utilizando Rational SoDA
En esta guía de la herramienta se describe cómo utilizar Rational SoDA para crear un documento de arquitectura de software.
Herramienta: Rational SoDA
Descripción principal

Visión general 

SoDA automatiza la generación del informe, de forma que se pueda crear rápidamente y con precisión. Puede generar un documento de arquitectura de software con la versión de Microsoft® Word® o Adobe® FrameMaker® de SoDA. Para crear el informe, SoDA recopila aspectos significativos arquitectónicamente de un modelo de Rational Rose. Sólo funciona si el modelo sigue la estructura y el convenio de denominación para el modelo de Rose.

Información relacionada con Rational Unified Process:

  • Informe: Inspección del modelo de diseño
  • Informe: Ejecución de guiones de uso
  • Producto de trabajo: Documento de arquitectura de software

Esta guía de la herramienta se aplica al ejecutar Windows 2000, NT 4.0, Windows XP, Solaris o HP-UX.

Para crear un documento de arquitectura de software utilizando SoDA, siga el procedimiento que se indica para su versión del producto:

  • Utilizar Rational SoDA/Word para generar un documento de arquitectura de software
  • Utilizar Rational SoDA/FrameMaker para generar un documento de arquitectura de software

Utilizar Rational SoDA/Word para generar un documento de arquitectura de software

  1. Desde cualquier lugar de Rational Rose, pulse Informe > Informe SoDA.
  2. Cuando aparezca la lista de informes disponibles en SoDA, seleccione Documento de arquitectura de software de Rational Unified Process.
  3. Pulse Aceptar para generar el informe.

Utilizar Rational SoDA/FrameMaker para generar un documento de arquitectura de software

  1. En la barra de botones de FrameMaker, pulse Nuevo. Efectúe una doble pulsación en SoDA y, a continuación, efectúe una doble pulsación en RoseDomain y elija la plantilla RUPSoftwareArchitectureDocument.fm.
  2. Edite el conector y entre el nombre del modelo.
  3. Pulse Archivo > Guardar como para guardar la plantilla en un directorio personal o en el directorio del proyecto.
    Es posible que desee cambiar el nombre de la plantilla de modo que refleje el nombre del guión de uso; por ejemplo, ConductTransactionsReport.fm.
  4. Pulse SoDA > Generar documento.
  5. Revise el documento generado.

La próxima vez que desee generar este documento, simplemente, abra el documento y pulse SoDA > Generar documento.

Estructura y convenio de denominación para el modelo de Rose

A continuación, se listan los diagramas que extrae SoDA del modelo de Rose para su inclusión en el documento de arquitectura de software:
  • Los guiones de uso y los actores que son significativos arquitectónicamente se muestran en la sección Vista de guión de uso del documento.
  • Las clases, las interfaces, los paquetes y los subsistemas que son significativos arquitectónicamente se muestran en la sección Vista lógica del documento.
  • Los paquetes que representan capas del modelo de diseño se muestran en la sección Vista lógica del documento.
  • Todos los diagramas del paquete Vista de proceso se muestran en la sección Vista de proceso del documento.
  • Todos los diagramas del paquete Modelo de implementación se muestran en la sección Modelo de implementación del documento.
  • Todos los diagramas de la Vista de despliegue se muestran en la sección Vista de despliegue del documento.
