Instrucciones de la herramienta: Ejecución de un conjunto de aplicaciones de prueba utilizando Rational TestManager
En esta guía de la herramienta se describe cómo utilizar Rational TestManager para ejecutar un conjunto de aplicaciones de prueba.
Herramienta: Rational TestManager
Relaciones

Descripción principal

Visión general

Como parte de la posibilidad de gestionar el esfuerzo de la prueba, TestManager le permite coordinar scripts de prueba de todos los tipos, es decir, GUI, prueba manual, usuario virtual, línea de mandatos, Virtual Basic y Java, en un paso ejecutado por el usuario. Este paso, denominado conjunto de aplicaciones, le permite distribuir, ejecutar y evaluar familias de scripts de prueba a través del equipo de prueba como un todo cohesivo, en lugar de en fragmentos.

Cuando ejecuta un conjunto de aplicaciones, proporciona información específica de tiempo de ejecución. Cada verificador virtual que ejecuta los elementos del conjunto de aplicaciones asignado se ejecuta con estas directrices. Los resultados de la ejecución del conjunto de aplicaciones se almacenan en registros. Después de ejecutar el conjunto de aplicaciones, puede generar informes para analizar los datos almacenados en los registros y visualizar resultados de la ejecución en formato de gráficos.

Para obtener información sobre cómo crear un conjunto de aplicaciones de prueba, consulte la Guía de la herramienta: Diseño de un conjunto de aplicaciones de prueba automatizado utilizando Rational TestManager.

Esta guía de la herramienta se aplica al ejecutar Microsoft Windows 98/2000/NT 4.0.

Pasos de la herramienta

Para ejecutar un conjunto de aplicaciones de prueba utilizando TestManager, debe llevar a cabo los pasos que se indican a continuación:

  1. Especificar un conjunto de aplicaciones
  2. Definir la carga de usuarios
  3. Especificar un archivo de registro
  4. Habilitar o inhabilitar la supervisión de recursos
  5. Elegir si se deben ignorar configuraciones
  6. Especificar opciones de informe de tiempo de ejecución

1.  Especificar un conjunto de aplicaciones

  • Para especificar un conjunto de aplicaciones, en el menú Archivo, pulse Ejecutar conjunto de aplicaciones. Se muestra el diálogo Ejecutar conjunto de aplicaciones.
  • Para cambiar el conjunto de aplicaciones, pulse Cambiar... en Información del conjunto de aplicaciones.

2.  Definir la carga de usuarios

  • Si el conjunto de aplicaciones que ha elegido ejecutar tiene definido un grupo de sistemas fijo y no existen campos visibles en Nombre, vaya al paso 3, Especificar un archivo de registro.
  • Si no es así, entre el número de verificadores virtuales que desea ejecutar en el recuadro Número de usuarios.
  • Si el conjunto de aplicaciones incluye tanto grupos fijos como escalables, primero se asignan los grupos fijos. El número de verificadores virtuales disponibles depende del tipo de licencia de la que disponga. Si la licencia no admite el número de verificadores virtuales que especifica, se muestra un mensaje de error.

Nota: Cuando define grupos de sistemas, si selecciona el recuadro de selección Solicitar sistemas antes de ejecutar el conjunto de aplicaciones, se muestra la lista Sistemas junto con los botones Editar sistemas, Editar listas de sistemas y Propiedades.

3.  Especificar un archivo de registro

El número de build actual se muestra en el recuadro Build. El nombre de la carpeta de registro se basa en el conjunto de aplicaciones, y el nombre del registro se basa en el número de verificadores virtuales y el número de veces que se ha ejecutado el conjunto de aplicaciones.

Para cambiar la carpeta de registro o el nombre de registro, pulse Cambiar... en Información de registro.

4.  Habilitar o inhabilitar la supervisión de recursos

Si prevé supervisar recursos, pulse el recuadro de selección Supervisar recursos. Modifique el intervalo del recuadro Frecuencia de actualización para cambiar la frecuencia de actualización de las vistas. Cuanto menor es el intervalo, más rápida es la actualización.

La supervisión observa el uso de recursos del sistema cuando se reproduce el conjunto de aplicaciones y, a continuación, se muestran gráficos sobre los datos de uso con respecto a los tiempos de respuesta de los verificadores virtuales correspondientes al analizar los resultados.

5.  Elegir si se deben ignorar configuraciones

Pulse el recuadro de selección Ignorar configuraciones para guiones de prueba para que TestManager ignore las configuraciones para guiones de prueba y ejecute los guiones de prueba en cualquier sistema disponible.

TestManager tiene tres modos de ejecutar conjuntos de aplicaciones con esta opción seleccionada:

  • Si un conjunto de aplicaciones contiene un guión de prueba con guiones de prueba configurados y el guión de uso padre tiene una implementación, por ejemplo, un script o conjunto de aplicaciones de prueba, TestManager ejecuta el guión de prueba padre en todos los sistemas disponibles, pero no ejecuta ninguno de los guiones de prueba configurados.
  • Si un conjunto de aplicaciones contiene un guión de prueba con guiones de prueba configurados y el guión de prueba padre no tiene ninguna implementación, TestManager no ejecuta el guión de uso padre ni ninguno de los guiones de uso configurados.
  • Si un único guión de prueba configurado tiene una implementación, TestManager ejecuta el guión de prueba en el sistema especificado.

6.  Especificar opciones de informe de tiempo de ejecución

  • Para cambiar el modo en el que TestManager informa sobre el estado de la ejecución del conjunto de aplicaciones, pulse Opciones.... Puede establecer los tipos de opciones siguientes:
    • La pestaña Supervisar controla el modo en el que TestManager muestra la información durante la ejecución del conjunto de aplicaciones.
    • La pestaña Informes especifica los informes que genera TestManager una vez que ha finalizado la ejecución del conjunto de aplicaciones. Esta opción sólo se aplica a la prueba de rendimiento.
    • La pestaña Ejecutar controla el registro de la ejecución del conjunto de aplicaciones para su posterior evaluación.
    • La pestaña Compilación VU especifica las definiciones por omisión para la compilación VU condicional y la inclusión de bibliotecas C externas en el conjunto de aplicaciones.

   Para obtener más información, consulte el tema de la ayuda en línea de Rational TestManager que trata sobre el recuadro de diálogo Opciones.

  • Una vez que haya configurado la ejecución, pulse Aceptar para ejecutar la prueba. TestManager comprueba el conjunto de aplicaciones y compila los scripts de prueba no compilados u obsoletos. Para detener la ejecución del conjunto de aplicaciones mientras TestManager lo comprueba, pulse Cancelar.
  • Cuando TestManager termina de comprobar el conjunto de aplicaciones y de compilar los scripts de prueba necesarios, minimiza la ventana Preparando para la ejecución. En este punto, no puede cancelar ni cerrar esta ventana. Para parar la ejecución, pulse Detener en el menú Supervisar.
  • Después de ejecutar las pruebas, puede evaluar los resultados de éstas.

   Para obtener más información, consulte el tema de la ayuda en línea de TestManager que trata sobre la ejecución de pruebas.
