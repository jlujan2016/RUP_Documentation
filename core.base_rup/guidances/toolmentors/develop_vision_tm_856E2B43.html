Instrucciones de la herramienta: Desarrollo de una visión utilizando Rational RequisitePro
En esta guía de la herramienta se describe cómo utilizar Rational RequisitePro® para documentar una visión del proyecto. El documento Visión es una sentencia general de los requisitos centrales del proyecto y proporciona la base contractual para los requisitos técnicos.
Herramienta: Rational RequisitePro

Descripción principal

Visión general

El documento Visión proporciona una base de alto nivel y, en ocasiones, contractual, para los requisitos técnicos. También puede haber una especificación de requisitos formal. La visión captura restricciones de diseño y requisitos de muy alto nivel a fin de que el lector comprenda el sistema que se va a desarrollar. Comunica los "qué y porqué" fundamentales relacionados con el proyecto y es un indicador contra el que se deben validar todas las decisiones futuras.

Pasos de la herramienta

Para documentar la visión utilizando RequisitePro, debe:

  1. Crear un documento Visión
  2. Crear requisitos en el documento Visión

1. Crear un documento Visión

El documento Visión contiene las características del producto identificadas en la Tarea: Deducir las solicitudes de los interesados. Si va a crear un proyecto de RequisitePro utilizando una de las plantillas de proyecto por omisión (guión de uso, tradicional o compuesta), el proyecto ya contiene un documento Visión en el paquete Características y visión. Si el paquete no contiene un documento Visión, siga el procedimiento que se indica a continuación para crear uno.

Para crear un documento de requisitos de Visión:

  1. Abra el proyecto de RequisitePro.
  2. En el Explorador, seleccione el paquete en el que desea situar el documento de Visión y, a continuación, pulse Archivo > Nuevo > Documento. Se muestra el recuadro de diálogo Propiedades del documento.
  3. Escriba un nombre, descripción y nombre de archivo para el documento.
  4. En la lista Tipo de documento, seleccione "Tipo de documento Visión". Pulse Aceptar. La esquematización para le documento Visión se abre en Microsoft® Word.
  5. Reemplace el texto de instrucciones por omisión por información específica del proyecto.
  6. Pulse RequisitePro > Documento > Guardar.

  • Creación de un proyecto de RequisitePro (Índice: proyectos>creación)
  • Creación de documentos de requisitos (Índice: documentos>creación)

2. Crear requisitos en el documento Visión

Cree requisitos a partir de las características del producto que se esquematizan en la sección Características del producto del documento Visión. Esto le permite gestionar atributos de requisitos en una Matriz de atributos y le ayuda a hacer un seguimiento de los enlaces de rastreabilidad entre características del producto y requisitos de software.

Para crear requisitos en el documento Visión:

  1. En la sección Características del producto del documento Visión, seleccione (resalte) el texto que define el requisito.
  2. Realice una de las acciones siguientes:
    • Pulse el botón derecho del ratón y seleccione Nuevo requisito.
    • Pulse RequisitePro > Requisito > Nuevo.
      Se muestra el recuadro de diálogo Propiedades del requisito.
  3. En la pestaña General, acepte el tipo de requisito de característica (FEAT) y escriba un nombre para el requisito.
  4. Pulse la pestaña Atributos y establezca el atributo Prioridad en uno de los valores proporcionados (Debe, Debería, Puede, No tendrá). Es recomendable registrar las prioridades de los interesados al deducir las solicitudes de requisitos de los interesados. (Consulte el apartado Tarea: Deducir las solicitudes de los interesados).
  5. Seleccione el atributo Procedencia y seleccione una entrada que represente al originador de la característica concreta. Esto le puede ayudar a hacer un seguimiento de quiénes solicitan características del producto específicas, a fin saber con quién contactar en caso de que necesite aclarar o negociar una solicitud.
  6. Pulse Aceptar.
  7. Puede repetir estos pasos para cada una de las características del producto que se esquematizan en la sección Características del producto del documento Visión. Una vez que haya finalizado, pulse RequisitePro > Documento > Guardar.

Para ver los requisitos y los atributos:

  1. En la ventana de RequisitePro, seleccione el paquete en el que desea crear una vista.
  2. Pulse Archivo > Nuevo > Ver. Se muestra el recuadro de diálogo Ver propiedades.
  3. Escriba un nombre para la vista, seleccione el tipo de vista Matriz de atributos y, a continuación, seleccione el tipo de requisito FEAT que ha utilizado en el documento Visión.
  4. Pulse Aceptar. Revise los requisitos de la característica y sus valores de atributo.
