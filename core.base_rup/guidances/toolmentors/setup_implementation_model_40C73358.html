Descripción principal

Terminología

Existen algunas diferencias entre la terminología de RUP y  la que se utiliza en ClearCase. Las definiciones siguientes de la terminología específica de la herramienta se proporcionan como ayuda para aclarar las diferencias.

  • Gestión de la configuración: En el contexto de ClearCase, la utilización del término Gestión de la configuración hace referencia al control de versión y a la gestión de compilaciones. RUP utiliza las definiciones IEEE e ISO de gestión de la configuración (CM), que también incluyen Gestión de cambios como parte de CM.
  • Gestión de cambios y de configuración: Tanto ClearCase como Rational ClearQuest hacen referencia a la gestión de cambios y de la configuración, que se sinónimo de la definición de RUP de gestión de la configuración. RUP utiliza estos términos con el mismo significado.
  • Proyecto: Un proyecto de ClearCase se correlaciona con el depósito de proyectos de RUP.

Visión general

En esta guía de la herramienta se describe cómo crear una infraestructura en el entorno de desarrollo para organizar y almacenar productos de trabajo de elementos de configuración relacionados con la implementación y la entrega del producto de software. La representación física de subsistemas en el entorno de desarrollo define la "estructura de directorios del producto" y, en realidad, es el modelo de implementación de software.

Antes de empezar a utilizar ClearCase, es importante que sepa si va a utilizar la Gestión unificada de cambios (UCM), un modelo de uso inmediato, o bien, ClearCase básico, que proporciona un conjunto de herramientas que se pueden utilizar para construir otros modelos de uso.

Esta guía de la herramienta se aplica a tareas de UCM de ClearCase y de ClearCase básico, mientras que todas las demás guías de la herramienta de ClearCase que se presentan en RUP utilizan el modelo UCM.

Esta guía de la herramienta se aplica al ejecutar Microsoft Windows.

Pasos de la herramienta

Para establecer el modelo de implementación con Rational ClearCase, debe:

  1. Crear depósitos de gestión de la configuración (CM) (VOB de ClearCase).
  2. Importar directorios y archivos existentes a las VOB.
  3. (Opcional) Crear una VOB administrativa para que contenga definiciones de objetos de tipo global.

1. Crear depósitos de gestión de la configuración (CM) (VOB de ClearCase)

Las VOB (bases de objetos con versión) almacenan versiones de archivos, directorios y otros objetos. Sirven como depósitos para información de gestión de la configuración. Es habitual crear las VOB para que sean representativas de los subsistemas que define la arquitectura.

Para iniciar el asistente para la creación de VOB (hay ayuda según contexto disponible en cada paso del asistente):

  1. En la barra de tareas de Windows, pulse Inicio > Programas > Software Rational > Rational ClearCase > Administración > Crear VOB.
  2. Se inicia el Asistente para la creación de VOB. Siga los pasos que se indican para crear una VOB de ClearCase. Antes de empezar, resulta útil conocer los convenios de denominación de VOB que se utilizan en el sitio o proyecto y dónde colocar el almacenamiento de la VOB.
  3. En los pasos siguientes del asistente, lleve a cabo lo siguiente:
    • Denomine la VOB e indique si debe ser un componente de UCM o una VOB de proyecto (PVOB).
    • Especifique una ubicación de almacenamiento de la VOB.
    • Elija Opciones.

  Para obtener detalles sobre la creación de VOB, consulte la ayuda en línea del asistente para la creación de VOB de ClearCase.

  Para obtener una visión general sobre el establecimiento de VOB, consulte el capítulo que trata sobre el establecimiento de VOB de ClearCase del manual Administrator's Guide de ClearCase.

2. Importar directorios y archivos existentes a las VOB

En una vista de ClearCase producida para el proyecto, cree la estructura de directorios de nivel superior que desea en las VOB, y copie directorios y archivos existentes en cada VOB para crear un conjunto inicial de elementos de configuración de desarrollo (elementos de Rational ClearCase).

  1. En el Explorador de ClearCase, seleccione los archivos y los directorios que desea importar. 
  2. Pulse el botón derecho del ratón para visualizar un menú de mandatos y seleccione Añadir al control de origen. De este modo, se crea un elemento de ClearCase para cada uno de los objetos resaltados.

  Para obtener información sobre la importación de grandes volúmenes de archivos, consulte el mandato clearimport en el manual Command Reference de ClearCase.

3. (Opcional) Crear una VOB administrativa para que contenga definiciones de objetos de tipo global

Si lo desea, puede crear una VOB administrativa para que contenga definiciones de objetos de tipo global, utilizados para definir líneas base en los subsistemas.

Para crear una VOB administrativa, inicie el asistente para la creación de VOB descrito en el Paso 1 y siga las indicaciones.

   Para obtener más información sobre cómo trabajar con VOB administrativas, consulte el apartado que trata sobre la utilización de VOB administrativas y tipos globales del manual de ClearCase titulado Administrator's Guide.
