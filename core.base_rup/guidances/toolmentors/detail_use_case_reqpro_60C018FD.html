Instrucciones de la herramienta: Detalle de un guión de uso utilizando Rational RequisitePro
This tool mentor describes how to use Rational RequisitePro® to describe a system use case in detail.
Herramienta: Rational RequisitePro

Descripción principal

Visión general

Una vez que se han identificado los casos de uso para el sistema propuesto (tal como se describe en la Guía de la herramienta: Búsqueda de actores y casos de uso de Rational Rose®), puede utilizar RequisitePro para desarrollar un documento de especificación del caso de uso .

El analista de sistemas realiza la descripción del caso de uso del sistema.

Nota: puede desarrollar los casos de uso en Rose y generarlos en RequisitePro utilizando la característica de Gestión de casos de uso integrados. Consulte la Guía de la herramienta: Gestión de casos de uso con Rational Rose y Rational RequisitePro para obtener más información.

Las secciones del documento de especificación del caso de uso se pueden utilizar para crear requisitos específicos. Los requisitos se pueden rastrear (o enlazar) a otros requisitos como, por ejemplo, características del producto.

Una persona que juega el rol de especificador de requisitos y es responsable de escribir una especificación del caso de uso para cada caso de uso , detalla la información textual para los casos de uso seleccionados. Este documento define todas las propiedades textuales del caso de uso y se puede ampliar con el nombre y la descripción del caso de uso producido en la tarea de RUP Buscar actores y casos de uso .

Pasos de la herramienta

Para detallar un caso de uso utilizando RequisitePro, debe:

  1. Crear un documento de especificación del caso de uso
  2. Completar el documento de especificación del caso de uso
  3. Crear requisitos en la especificación del caso de uso detallada
  4. Añadir diagramas a la especificación del caso de uso (opcional)

Si ha creado el proyecto de RequisitePro utilizando la plantilla de caso de uso , el proyecto ya contiene un tipo de documento de especificación del caso de uso .

1. Crear un documento de especificación del caso de uso

El documento de especificación del caso de uso contiene las propiedades textuales del caso de uso , lo que incluye las propiedades del caso de uso siguientes: nombre, descripción breve, flujo de suceso básico, flujo de sucesos alternativo, condiciones previas, condiciones posteriores y requisitos especiales.

Nota: si ha desarrollado los casos de uso en Rose, puede utilizar los procedimientos que se describen en  Guía de la herramienta: Gestión de casos de uso utilizando Rational Rose y Rational RequisitePro para crear un nuevo documento de caso de uso asociado al caso de uso de Rose. Si no es así, siga los pasos que se indican a continuación para crear un documento de caso de uso .

Para crear un documento de especificación del caso de uso en RequisitePro:
  1. En el Explorador, seleccione el paquete en el que desea que resida el nuevo documento y, a continuación, pulse Archivo > Nuevo > Documento. Se muestra el recuadro de diálogo Propiedades del documento.
  2. Escriba un nombre, descripción y nombre de archivo para el documento.
  3. En la lista Tipo de documento, seleccione "Tipo de documento de especificación del caso de uso ".
  4. Pulse Aceptar. La esquematización del documento de especificación del caso de uso se abre en Microsoft® Word.

Para obtener más información

Consulte el tema de la ayuda en línea de RequisitePro que trata sobre la creación de documentos de requisitos (Índice: documentos>creación).

2. Completar el documento de especificación del caso de uso

En el documento de especificación del caso de uso recién creado, escriba información específica del caso de uso en cada sección. Las propiedades de nombre y descripción breve ya se deben haber documentado al identificar originalmente el caso de uso .

Para completar el documento de especificación del caso de uso :

  1. Reemplace el texto "Nombre de caso de uso " que se muestra en la esquematización por el nombre real del caso de uso .

    Nota: si ha creado el documento del caso de uso utilizando los procedimientos que se describen en la Guía de la herramienta: Gestión de casos de uso utilizando Rational Rose y Rational RequisitePro, el nombre del caso de uso se inserta automáticamente en el título del documento. Utilice los mandatos RequisitePro > Requisito > Cortar y Pegar para mover el requisito del caso de uso al texto "Nombre de caso de uso ".

  2. Lea las instrucciones que se indican en la sección Descripción breve y, a continuación, suprímalas y escriba una descripción breve.

    Nota: si ha desarrollado el caso de uso en Rose y desea incluir el campo de documentación de Rose como parte de la sección de descripción breve en el documento del caso de uso de RequisitePro, copie el texto del campo Documentación en el recuadro de diálogo Especificación del caso de uso de Rose y péguelo en el documento de especificación del caso de uso .

  3. Reemplace el texto por omisión que se encuentra en la sección Flujo de sucesos básico por el texto correspondiente al flujo de sucesos básico de este caso de uso . Identifique cada paso en una línea separada.

  4. Repita este procedimiento para las demás propiedades del caso de uso (flujo de sucesos alternativo, requisitos especiales, condiciones previas y condiciones posteriores, entre otras).

  5. Pulse RequisitePro > Documento > Guardar.

Para obtener más información

Consulte el tema de la ayuda en línea de RequisitePro que trata sobre cómo guardar documentos de requisitos (Índice: documentos>cómo guardar).

3. Crear requisitos en la especificación del caso de uso detallada

Con el objeto de hacer un seguimiento de los enlaces de rastreabilidad entre casos de uso y otras informaciones como, por ejemplo, el modelo de caso de uso o características del producto, cree requisitos de la secciones de especificación del caso de uso . Convierta el nombre del caso de uso en un requisito padre y las propiedades en requisitos hijo. Las propiedades pueden incluir descripciones breves, acciones en los flujos de sucesos básico o alternativo, condiciones previas, condiciones posteriores, requisitos especiales y relaciones ampliadas.

  1. En el documento de especificación del caso de uso , seleccione el nombre del caso de uso .

  2. Realice una de las acciones siguientes:

    • Pulse el botón derecho del ratón y seleccione Nuevo requisito.
    • Pulse RequisitePro > Requisito > Nuevo.
      Se muestra el recuadro de diálogo Propiedades del requisito.

  3. Seleccione UC: Tipo de requisito del caso de uso para el tipo de requisito.

  4. En la pestaña Atributos, en el atributo Propiedad seleccione el valor "Nombre" de la lista de valores.

  5. Repita los pasos anteriores para la descripción breve (estableciendo el atributo Propiedad en "Descripción breve"). En la pestaña Jerarquía, seleccione <elegir padre> e identifique el requisito UC que representa el nombre del caso de uso .

  6. En la sección de flujo de sucesos básico del documento de especificación del caso de uso , cree requisitos UC para cada paso o grupo de pasos (subflujo) al que desea establecer enlaces de rastreabilidad. Establezca el atributo Propiedad en Flujo básico, y establezca el padre del requisito en el requisito de nombre del caso de uso que ha creado en los Pasos 1 a 3 anteriores. Se le proporciona la opción de indicar grupos de pasos que siempre se realizan juntos. Si fuera necesario, utilice requisitos jerárquicos para distinguir subflujos del flujo de sucesos básico.

    Nota: no es necesario que cree requisitos para cada paso de un flujo de sucesos. El valor de crear requisitos de flujo de sucesos está en rastrear de un requisito de nivel superior como, por ejemplo, una característica del producto, a una parte específica del flujo de sucesos. Con frecuencia, un flujo de sucesos abarca varias páginas, por lo que quizá sea preferible considerar el flujo de sucesos en conjunto.

  7. En cada flujo de sucesos alternativo, cree requisitos UC para cada paso o grupo de pasos (subflujo) al que desea establecer enlaces de rastreabilidad. Establezca el atributo Propiedad en Flujo alternativo y el requisito padre tal como se ha indicado previamente. Del modo similar al flujo de suceso básico, utilice requisitos jerárquicos para indicar subflujos completos.

  8. Los pasos siguientes son opcionales:

    • En la sección de condiciones previas del documento de especificación del caso de uso , seleccione cada condición previa por separado y cree un requisito UC (Propiedad = condiciones previas, padre = requisito de nombre del caso de uso ).
    • Repita los pasos anteriores para las condiciones posteriores (Propiedad = condiciones posteriores) y la sección de requisitos especiales (Propiedad = Especial). Establezca el requisito de nombre del caso de uso como padre.

Para obtener más información

Consulte el tema de la ayuda en línea de RequisitePro que trata sobre la creación de requisitos en un documento (Índice: requisitos>creación).

4. Añadir diagramas a la especificación del caso de uso (opcional)

Algunas de las propiedades del caso de uso son no textuales como, por ejemplo, "diagramas de caso de uso " y "otros diagramas". Consulte el Producto de trabajo: Caso de uso de RUP. Estos diagramas se almacenan en Rose. Gracias a Rational SoDA®, puede crear un informe de caso de uso a partir de las propiedades textuales del caso de uso almacenadas en RequisitePro y la información de diagrama de caso de uso almacenada en Rose.
