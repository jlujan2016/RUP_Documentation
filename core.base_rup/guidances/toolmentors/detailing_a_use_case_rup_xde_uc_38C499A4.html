Visión general 

Los pasos de esta guía de la herramienta coinciden con los de la tarea. Los enlaces a los temas de la ayuda en línea de Rational XDE ™ están marcados con .

En esta guía de la herramienta se da por supuesto que se ha creado un modelo de caso de uso de acuerdo con las directrices de la estructura de modelo de XDE, y también se presupone que se ha creado un caso de uso siguiendo los pasos que se indican en la Guía de la herramienta: Búsqueda de actores y casos de uso utilizando Rational XDE.

En esta guía de la herramienta se llevan a cabo los pasos siguientes:

  • Detallar el flujo de sucesos del caso de uso
  • Estructurar el flujo de suceso del caso de uso
  • Ilustrar relaciones con actores y otros casos de uso <opcional>
  • Describir los requisitos especiales del caso de uso
  • Describir protocolos de comunicación
  • Describir condiciones previas del caso de uso <opcional>
  • Describir condiciones posteriores del caso de uso <opcional>
  • Describir puntos de ampliación <opcional>
  • Evaluar los resultados

Antes de detallar un caso de uso , debe decidir cómo desea capturar los detalles. Algunas opciones son:

  • Utilizar la ventana Documentación de modelo, que le permite documentar cualquier elemento de modelo seleccionado. Consulte el apartado Documentación de elementos de modelo
  • Se puede crear un documento de especificación de caso de uso separado utilizando la plantilla que proporciona RUP (consulte las plantillas del Producto de trabajo: Caso de uso ). A continuación, el documento se puede asociar al elemento de modelo de caso de uso . Consulte el apartado Enlace de archivos externos a elementos de modelo ;
  • También puede asociar los casos de uso con requisitos y documentos mediante la integración de Rational RequisitePro y XDE. Para asociar un caso de uso a un requisito o documento de RequisitePro, consulte la ayuda de la integración de RequisitePro y XDE. Puede acceder a dicha ayuda desde la barra de menús superior, pulsando Herramientas > Rational RequisitePro > Ayuda de la Integración. A continuación se facilita más información sobre la integración de RequisitePro y XDE.

Integración de Rational RequisitePro y XDE

Mediante la integración de Rational RequisitePro y XDE, puede editar los elementos del modelo de caso de uso desde sus requisitos asociados en las bases de datos y los documentos de RequisitePro. Puede asociar los casos de uso con documentos de requisitos de RequisitePro, lo que permite desarrollar una definición textual del caso de uso en un documento RequisitePro de Microsoft Word. En los documentos de caso de uso de RequisitePro, puede elaborar el caso de uso con descripciones, flujos de sucesos, requisitos especiales y condiciones.

La integración también le permite asignar atributos de requisitos, como la rastreabilidad, para modelar los elementos. Puede utilizar los requisitos asociados para rastrear las relaciones entre los elementos de modelo y otros requisitos mediante las capacidades de rastreabilidad de requisitos de RequisitePro. Si asocia los elementos de modelo con los requisitos, le permite revisar y valorar el impacto de los cambios de requisitos en los elementos de modelo.

Para utilizar la integración, RequisitePro debe estar instalado, y el perfil de RequisitePro debe aplicarse al modelo XDE. El perfil de RequisitePro se aplica automáticamente a un modelo creado con la plantilla de modelo de caso de uso . Para los modelos existentes, debe aplicar este perfil manualmente. Consulte el apartado Aplicación de perfiles a un modelo .

Detallar el flujo de sucesos del caso de uso

  1. Abra el modelo de caso de uso .
  2. Navegue al caso de uso que desea detallar.
  3. Entre un flujo de sucesos detallado en la documentación asociada con el elemento de modelo.

Estructurar el flujo de sucesos del caso de uso

Puede ilustrar la estructura del flujo de sucesos con un diagrama de actividad. Para ello, siga los pasos que se indican a continuación:

  1. Pulse el botón derecho sobre el caso de uso en el Explorador de modelos, y añada un diagrama de actividad. Consulte el apartado Adición de diagramas .
  2. (opcional) Añada divisiones. Consulte el apartado Adición de formas a diagramas .
  3. Añada el estado inicial, el estado final y las actividades. Consulte el apartado Adición de formas a diagramas .
  4. Anote cada actividad con una descripción breve. Consulte el apartado Documentación de elementos de modelo .
  5. Conecte los estados de actividad con transiciones. Consulte el apartado Adición de relaciones de transición .
  6. Cree barras de sincronización (según sea necesario). Consulte el apartado Adición de formas a diagramas .
  7. Especifique condiciones de vigilancia en las transiciones (según sea necesario). Consulte el apartado Especificar condiciones de vigilancia en relaciones de transición .

Ilustrar relaciones con actores y otros casos de uso <opcional>

Este es un paso opcional que se lleva a cabo únicamente para aclarar la relación entre el caso de uso y sus actores y casos de uso asociados. La adición de relaciones con actores se describe en Guía de la herramienta: Búsqueda de actores y casos de uso utilizando Rational XDE. Las relaciones con otros casos de uso se describen en Guía de la herramienta: Estructurar el modelo de caso de uso utilizando Rational XDE.
