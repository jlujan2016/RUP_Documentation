Instrucciones de la herramienta: Configuración de proyectos utilizando Rational Administrator
En esta guía de la herramienta se describe cómo utilizar Rational Administrator para configurar proyectos del administrador.

Descripción principal

Visión general

En el administrador, puede crear proyectos del administrador que almacenan información de desarrollo y pruebas de software. Todos los componentes de Rational del sistema actualizan y recuperan datos del mismo proyecto. Los tipos de datos de un proyecto del administrador dependen del software Rational que se haya instalado. 

Para obtener información sobre los proyectos del administrador, consulte los temas de la ayuda en línea del administrador que tratan sobre:

  • ¿Qué es un proyecto de Rational?
  • Partes de un proyecto de Rational
  • Ventajas de un proyecto de Rational

Gracias al administrador, un proyecto del administrador también se puede habilitar para UCM (Gestión unificada de cambios), que es la propuesta del software Rational para gestionar el cambio de requisitos, diseñar modelos, documentación, componentes, guiones de prueba y código fuente, entre otros.  

Para obtener información sobre UCM, consulte los temas de la ayuda en línea del administrador que tratan sobre:

  • ¿Qué es UCM?
  • ¿Qué es UCM en relación con Rational Administrator?
  • Cómo decidir si convertir el proyecto en habilitado para UCM

Antes de configurar un proyecto, lo debe crear, y antes de crear un proyecto del administrador, debe resolver cuestiones clave. Si prevé habilitar el proyecto para UCM, también debe configurar Rational ClearCase. 

Para obtener información sobre los pasos que debe seguir, incluida la creación de un proyecto del administrador, consulte los temas de la ayuda en línea del administrador que tratan sobre: 

  • Antes de crear un proyecto
  • Pasos para la configuración de ClearCase
  • Creación de un proyecto de Rational Administrator

Si desea utilizar depósitos existentes, los puede convertir en proyectos del administrador, de modo que los pueda utilizar la versión actual del administrador.  

Para obtener más información sobre la conversión, consulte los temas de la ayuda en línea del administrador que tratan sobre: 

  • Actualización de un depósito SQA 6.x
  • Actualización de un depósito de Rational Test 7.x

Esta guía de la herramienta se aplica al ejecutar Windows 98/2000/NT 4.0.

Para configurar un proyecto del administrador debe:

  1. Decidir las estructuras de datos de Rational que desea incluir en el proyecto
  2. Decidir si desea asociar el proyecto del administrador a un proyecto de UCM (si aún no lo ha hecho)
  3. Decidir si desea que un proyecto de Rational RequisitePro o un almacén de datos de Rational Test se encuentren bajo la gestión de la configuración (si aún no lo ha hecho)

1.  Decidir las estructuras de datos de Rational que desea incluir en el proyecto

Puede asociar distintas estructuras de datos al proyecto del administrador. Las estructuras de datos son las siguientes:

  • Almacenes de datos de Rational Test
  • Proyectos de Rational RequisitePro
  • Bases de datos de Rational ClearQuest
  • Archivos de modelo de Rational Rose

Puede crear las estructuras de datos desde el administrador o, si lo desea, puede seleccionar estructuras de datos existentes, excepto archivos de modelo de Rose. Los archivos de modelo de Rose sólo se pueden añadir, no crear, desde el administrador.

  Consulte el tema que trata sobre la configuración de un proyecto de Rational Administrator de la ayuda en línea del administrador.

2.  Decidir si desea asociar el proyecto del administrador a un proyecto de UCM

Una vez que haya creado un proyecto del administrador, puede habilitar el proyecto para UCM si lo había creado como no habilitado para UCM, pero sólo si ha instalado Rational ClearCase en el sistema. También puede habilitar activos de prueba y activos de requisitos para UCM en el proyecto como parte de la modificación de las propiedades del proyecto.

Tenga en cuenta que no puede inhabilitar un proyecto habilitado para UCM que tenga almacenes de datos en UCM, lo que significa que no puede cambiar el estado de un proyecto habilitado para UCM a no habilitado para UCM, a menos que los almacenes de datos de requisitos y prueba también estén como no habilitados para UCM.

   Consulte el tema que trata sobre la asociación de un proyecto de Rational Administrator a un proyecto de UCM de la ayuda en línea del administrador.

3.  Decidir si desea que un proyecto de Rational RequisitePro o un almacén de datos de Rational Test se encuentren bajo la gestión de la configuración

Si ya ha asociado un proyecto de RequisitePro o un almacén de datos de Test a un proyecto que está habilitado para UCM, pero aún no se encuentran bajo la gestión de la configuración, lo puede hacer.

Además, cualquier proyecto de UCM que se asocie al proyecto del administrador se habilita para la gestión de solicitudes de cambio si se selecciona una base de datos de ClearQuest en el recuadro de diálogo Configurar proyecto.

No obstante, si ya se ha asociado una base de datos de ClearQuest a un proyecto de UCM, no se puede seleccionar ni crear otra base de datos de ClearQuest para asociarla al proyecto del administrador. Para crear o cambiar la base de datos de ClearQuest asociada al proyecto del administrador, debe crear un nuevo proyecto. La base de datos de ClearQuest se predetermina en función de las relaciones existentes con un proyecto de UCM.

En el proyecto de administrador habilitado para UCM, puede separar un proyecto de RequisitePro o un almacén de datos de Test que se encuentre bajo la gestión de la configuración. Tenga en cuenta que al eliminar un proyecto de RequisitePro o un almacén de datos de Test de la gestión de la configuración sucede lo siguiente:

  • Se copian todos los archivos del proyecto de RequisitePro y del almacén de datos de Test a un área no controlada por la versión
  • Se asocia el proyecto al área no controlada por la versión

   Consulte el tema que trata sobre la configuración de un proyecto de Rational Administrator de la ayuda en línea del administrador.
