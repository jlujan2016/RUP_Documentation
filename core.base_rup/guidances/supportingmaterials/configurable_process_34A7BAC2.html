Descripción principal

Rational Unified Process ofrece una infraestructura que puede personalizarse para ajustarse a las necesidades específicas de cada empresa de desarrollo de software. Entre los factores que afectan a la personalización se encuentran la tecnología que se va a utilizar, las herramientas y los procesos que se usan en la empresa.

Rational Unified Process (RUP) es lo suficientemente general y completo como para ser utilizado por una amplia gama de empresas de desarrollo de software. En numerosas circunstancias, este proceso de ingeniería de software deberá modificarse, ajustarse, ampliarse y personalizarse para adaptarse a las características, restricciones e historial específicos de la empresa.

La disciplina de entorno describe los procedimientos de personalización e implementación de un nuevo proceso de desarrollo de software en un proyecto. El resultado de esta personalización se refleja en el proceso específico del proyecto. Consulte Concepto: Personalización de RUP para obtener detalles.

Puede ampliar el sitio web de RUP para incorporar sus conocimientos del proceso de la empresa de desarrollo o un conjunto de activos reutilizables. Para obtener más información, consulte Rational Method Composer.
