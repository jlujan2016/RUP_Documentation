Lista de comprobación: Caso de prueba
Esta lista de comprobación le ayuda a garantizar que todos los casos de prueba se han especificado correctamente y que tratan todos los requisitos de prueba.

Descripción principal
Elementos de comprobación
Una descripción del caso de uso, del caso de ejemplo de caso de uso, del objetivo de prueba o de las condiciones se han indicado correctamente para cada caso de prueba
Cada caso de prueba indica el resultado esperado y el método para evaluar el resultado
Para cada requisito de prueba, se han identificado como mínimo dos casos de prueba
Un caso de prueba, que representa una condición esperada, se desarrolla para verificar el comportamiento correcto o esperado (prueba positiva). Un segundo caso de prueba, que representa una condición inaceptable, anormal, o inesperada, se desarrolla para verificar que el requisito de la prueba no se ejecuta de forma inesperada (prueba negativa). Habitualmente, para cada requisito de prueba, existirá como mínimo un caso de prueba positivo y muchos casos de prueba negativos.
Los casos de prueba se han identificado para ejecutar todos los comportamientos de requisito de los productos en el destino de la prueba

Deben incluir (según corresponda)

  • función
  • validación de datos
  • implementación de reglas empresariales
  • flujo de trabajo o control del destino de la prueba
  • rendimiento del estado del objeto de flujo de datos (incluyendo carga de trabajo, configuración y tensión)
  • seguridad y accesibilidad
  • cumplimiento
Cada caso de prueba describe o representa un conjunto exclusivo de entradas o secuencia de sucesos cuyo resultado sea el comportamiento exclusivo del destino de la prueba
Revise los casos de prueba que producen el mismo comportamiento y determine si son equivalentes, es decir, que ambos ejecutan la vía de acceso en el destino de la prueba.
En cada caso de prueba, o grupo de casos de prueba relacionados, se identifica el estado inicial del destino de la prueba y el estado de los datos de prueba
Todos los nombres de caso de prueba y/o ID son coherentes con el convenio de denominación de producto de trabajo de prueba
