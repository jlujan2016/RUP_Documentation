Descripción principal
 
Elementos de comprobación
El nombre de la clase de análisis es exclusivo
La clase se utiliza en una colaboración por lo menos
La breve descripción de la clase captura la finalidad de la clase y resume brevemente sus responsabilidades.
La clase representa un único conjunto de responsabilidades coherentes
Los nombres de las responsabilidades son descriptivos y las descripciones de las responsabilidades son correctas
Las responsabilidades de la clase son coherentes con las expectativas que ha generado por las colaboraciones en las que participa la clase
Todas las clases que deben realizar los guiones de uso (excepto las clases de diseño) se han identificado
Todas las interacciones entre el actor y el sistema tienen el soporte de alguna clase de límite
No hay dos clases que posean la misma responsabilidad
Cada clase de análisis representa un conjunto distinto de responsabilidades, que es coherente con la finalidad de la clase
Las relaciones entre los guiones de uso (inclusión, ampliación, generalización) se manejan de forma coherente en el modelo de análisis
Se especifica el ciclo vital completo (creación, uso, eliminación) de cada clase de análisis
La clase cumple con las responsabilidades que se le exigen, sea de forma directa o mediante delegación
Las colaboraciones de clases tienen soporte de las asociaciones pertinentes
Todos los requisitos de la clase se han abordado
Si la clase es una clase de límite, todos los requisitos del actor se han abordado (incluido el error de entrada)
