Descripción principal


Cuando se evalúa el modelo de análisis de carga de trabajo, también es una buena idea asegurarse de que los planes se encuentran en su lugar para probar el sistema contra los requisitos de rendimiento, y que existen planes para validar las estimaciones de rendimiento del sistema utilizando prototipos arquitectónicos, especialmente para requisitos críticos de rendimiento.



Elementos de comprobación
Todos los requisitos de rendimiento nominales y máximos se especifican
Los requisitos de rendimiento son razonables y son un reflejo de las restricciones reales del dominio del problema
su especificación no es arbitraria.
El modelo de análisis de carga de trabajo proporciona estimaciones de rendimiento del sistema que indican qué requisitos de rendimiento, si los hubiere, son riesgos
Los 'objetos de cuello de botella' se han identificado y se han definido estrategias para evitar la aparición de cuellos de botella
Las cuentas de mensaje de colaboración son adecuadas visto el dominio del problema
Las colaboraciones deben estar bien organizadas al máximo y deben ser lo más sencillas posible.
El inicio ejecutable (inicialización) se encuentra dentro de los límites aceptables tal como se define en los requisitos
