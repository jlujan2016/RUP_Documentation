Lista de comprobación: Modelo de datos
Esta lista de comprobación le ayuda a garantizar que el modelo de datos es correcto y completo.

  • Modelo de datos
Descripción principal
Elementos de comprobación
Todas las clases permanentes que utilizan la base de datos para la permanencia se han correlacionado con las estructuras de la base de datos
Las relaciones de varios a varios tienen una tabla de intersección
Las claves principales se han definido para cada tabla, a menos que haya una razón de rendimiento por la que no se deba definir una clave principal
El almacenamiento y la recuperación de los datos se ha optimizado
Si se utiliza una base de datos relacional, las tablas se han desnormalizado (donde sea necesario) para mejorar el rendimiento.
Cuando se ha utilizado la desnormalización, todos los casos de ejemplo de actualización, inserción y supresión se han considerado para garantizar que la desnormalización no afecta negativamente al rendimiento de esas operaciones.
Los índices se han definido para optimizar el acceso
El impacto de las actualizaciones del índice se han considerado en las operaciones de la otra tabla
La distribución de datos se ha planeado
Se han definido restricciones de integridad referencial y de datos
Existe un plan para mantener las restricciones de validación cuando cambian las reglas de datos
Se han definido los desencadenantes y los procedimientos almacenados
El mecanismo de permanencia utiliza procedimientos almacenados y desencadenantes de bases de datos de forma coherente
