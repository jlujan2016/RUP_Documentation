Directriz: Diagrama de componentes
Un diagrama de componentes muestra la estructura de los componentes, incluyendo clasificadores que especifican componentes, y artefactos que los implementan. En esta directriz se identifican las situaciones en que crear estos diagramas puede resultar útil.
Relaciones
Elementos relacionados
  • Modelo de implementación
Descripción principal

Explicación

Los diagramas de componentes muestran la estructura de componentes, incluyendo los clasificadores que especifican componentes, y los artefactos que los implementan.

También se pueden utilizar para mostrar la estructura de alto nivel del modelo de implementación en términos de subsistemas de implementación, y las relaciones entre elementos de implementación.

Utilización

El uso más importante del diagrama de componentes de RUP es mostrar la estructura de nivel elevado del modelo de implementación. Específicamente:

  • Subsistemas de implementación y sus dependencias de importación. Consulte el apartado Directriz de producto de trabajo: Dependencia de importación en implementación.
  • Los subsistemas de implementación organizados en capas.

Un uso secundario del diagrama de componentes se muestra en:

  • Archivos de código fuente y sus dependencias de compilación. Consulte el apartado Directriz de producto de trabajo: Dependencia de compilación en implementación.
  • Archivos de aplicación y sus dependencias de tiempo de ejecución.
  • La derivación de relaciones entre archivos de código fuente y los archivos que resultan de compilaciones o enlaces.
  • Implementar dependencias entre elementos de implementación y los elementos de diseño que implementan. Consulte el apartado Directriz de producto de trabajo: Dependencia de manifiesto.

Tenga en cuenta que RUP enfatiza el uso del diagrama de componentes para modelar los subsistemas de implementación, los elementos de implementación significativos, y sus relaciones. Sin embargo, un diagrama de componentes que muestra componentes UML y los clasificadores que especifican estos componentes, también se pueden utilizar en el modelo de diseño, mediante el componente UML para representar el artefacto: subsistema de diseño. Consulte el apartado Directriz de producto de trabajo: Subsistemas de diseño para obtener más información sobre este tema.
