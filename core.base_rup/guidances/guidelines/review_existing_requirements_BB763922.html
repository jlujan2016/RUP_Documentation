Directriz: Revisión de los requisitos existentes
Esta directriz proporciona consejos para la revisión efectiva de los requisitos.

Descripción principal

Es posible que tenga especificaciones de requisitos de sistemas anteriores o de sistemas relacionados como referencia. Revisarlos puede resultar útil. O bien, es posible que haya empezado a utilizar Rational Unified Process después de empezar el proyecto.

Con el grupo, repase cada requisito para detectar comportamientos de la aplicación o atributos de comportamiento. En general, durante el ensayo, debe omitir la información explicativa como introducciones y descripciones generales del sistema.

Haga una lista con todas las cuestiones que identifique y asegúrese de que alguien se encarga de resolver cada cuestión. Es posible que necesite hacer algunas suposiciones si un requisito no está claro. Realice el seguimiento de las suposiciones para poder verificarlas con los interesados.

Recuerde quien escribió los requisitos. Busque los posibles "requisitos mal colocados", lo que significa que las cosas están fuera del ámbito del proyecto. Si no sabe si algo es un requisito, pregunte a los interesados.

Es muy eficaz llevar a cabo este tipo de ensayos mediante las esquematizaciones existentes del guión de uso como infraestructura. Cada requisito necesita relacionarse con, como mínimo, un guión de uso en la esquematización. Si no hay ningún guión de uso con el que relacionarse, es una indicación de que falta un guión de uso o que el requisito está mal colocado.
