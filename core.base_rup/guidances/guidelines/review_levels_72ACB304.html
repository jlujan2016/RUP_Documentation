Descripción principal

Utilice estos niveles de revisión para describir cómo revisar cada producto de trabajo. 

Niveles de revisión

Explicación

Comentarios

Formal-Externo

Este producto de trabajo forma parte de la entrega en un punto específico y requiere algún tipo de aprobación por parte del cliente, el patrocinador u otro interesado externo. El registro de revisión está gestionado por configuración junto con el producto de trabajo. 

Por ejemplo, la Visión y el Caso de negocio son productos de trabajo que normalmente deben revisar los interesados. 

Formal-Interno

Este producto de trabajo se revisa formalmente de forma interna por el proyecto. El registro de revisión está gestionado por configuración junto con el producto de trabajo.

Por ejemplo, las interfaces de subsistemas de diseño habitualmente tienen que revisarlas y aprobarlas diferentes miembros del proyecto. 

Informal

Este producto de trabajo se revisa, pero no se aprueba formalmente. El producto de trabajo se desarrolla y se mantiene. Normalmente no se descarta una vez que el proyecto se ha completado y no hay ningún registro de revisión. Por ejemplo, las clases de diseño individuales y el código de origen de bajo nivel no se puede revisar formalmente. Esto puede significar que alguien, quizás un igual, las revisará. 

Ninguno

Este producto de trabajo no necesita revisión ni aprobación. El producto de trabajo se ha creado como información de trabajo. Es, a menudo, un producto de trabajo temporal que se descarta una vez que el proyecto se ha completado.
