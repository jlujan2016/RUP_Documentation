Directriz: Guión gráfico
Un guión gráfico es una descripción lógica y conceptual de la funcionalidad del sistema para un caso de ejemplo específico. Esta directriz proporciona formas alternativas de capturar guiones gráficos.

Descripción principal

Introducción

Los guiones gráficos se pueden utilizar para explorar, comprender y razonar sobre los requisitos de comportamiento del sistema, especialmente, cómo interactuarán los usuarios con el sistema. Los guiones gráficos se utilizan desde hace tiempo en el cine y la televisión - de ellos ha tomado prestada la técnica la comunidad de software. Los guiones gráficos pretenden representar casos de ejemplo textuales más reales utilizando medios gráficos para especificar los requisitos. No pretenden ser un primer esbozo de la interfaz de usuario, sino que pretenden representar las interacciones del usuario con el sistema.

En esta directriz describimos algunas recomendaciones para representar el guión gráfico. Para obtener más información sobre los guiones gráficos, consulte el apartado Técnica: Guiones gráficos.

Representación de guiones gráficos

Los guiones gráficos pueden ser prototipos formales o informales, ejecutables o no ejecutables, de baja fidelidad (dibujados a mano) o de alta fidelidad (páginas HTML interactivas). El formato del guión gráfico no es relevante. Lo que es importante es recordar el objetivo del guión gráfico (comprender las expectativas del usuario respecto al comportamiento del sistema) y qué habilidades son necesarias para producir el guión gráfico (un guión gráfico requiere habilidades de adquisición de requisitos, no habilidades de diseño de interfaz de usuario).

Los guiones gráficos se pueden expresar con representaciones visuales o textuales, o una combinación de ambas.

Algunos ejemplos de los modos en que se pueden visualizar los guiones gráficos incluyen los siguientes:

  • Esbozos o dibujos en papel
  • Mapas de bits de una herramienta de dibujo
  • Tarjetas de índice
  • Diapositivas de Powerpoint
  • Capturas de pantalla (si existe una interfaz de usuario, o un prototipo de la interfaz de usuario)
    Nota: los guiones gráficos expresados en términos de capturas de pantalla reales pueden resultar una entrada útil en la documentación de usuario final.

No importa qué representación se seleccione, es importante tener en cuenta lo siguiente para cada elemento de la interfaz de usuario:

  • Las acciones que el usuario puede desarrollar, o las solicitudes que puede efectuar al sistema.
  • La información que está presente para el usuario, o la que ha entrado.
