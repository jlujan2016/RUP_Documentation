Directriz: Decisiones importantes de configuración y gestión de cambios
En esta directriz se describen cuestiones importantes a tener en cuenta cuando se personalizan los aspectos de Configuración y Gestión de cambios del proceso.
Relaciones
Elementos relacionados
  • Desarrollar el guión de desarrollo
  • Personalizar el proceso de desarrollo para el proyecto
  • Configuración y Gestión de cambios
Descripción principal

Decidir cómo utilizar los productos de trabajo

Decida qué productos de trabajo utilizar y cómo utilizar cada uno de ellos. En la tabla siguiente se describen los productos de trabajo que debe tener y los que se utilizan en algunos casos. Para obtener información más detallada sobre cómo personalizar cada producto de trabajo, y una discusión de las ventajas y desventajas de este producto de trabajo específico, lea la sección titulada "Personalización" de cada producto de trabajo.

Para cada producto de trabajo, decida cómo debe utilizarse el producto de trabajo:Debe tener, Debería tener, Puede tener o No tendrá. Para obtener más detalles, consulte el apartado Técnica: Clasificación de productos de trabajo.

Producto de trabajo Objetivo Personalización (Opcional, recomendada)
Solicitud de cambio Se utiliza para realizar el seguimiento de solicitudes de cambio de producto de trabajo de proyecto, incluyendo los defectos. Recomendada para la mayoría de proyectos.
Configuración de descubrimientos de auditoría Se utiliza para registrar los resultados de una auditoría de configuración. Opcional.

Generalmente se asocia a procesos más formales.
Plan de gestión de la configuración Describe todas las tareas de Configuración y de Gestión de control de cambios (CCM) que se llevarán a cabo durante el ciclo de vida del producto o proyecto. Opcional
Depósito de proyecto Almacena todas las versiones de archivos de proyecto y directorios. Recomendado.

Un sistema de gestión de la configuración para realizar el seguimiento de los archivos y compilaciones se recomienda para todos los proyectos.
Espacio de trabajo Proporciona un área de desarrollo privado dentro de la cual un miembro del equipo puede realizar cambios a los productos de trabajo sin que los cambios sean visibles inmediatamente para los demás. Recomendado para la mayoría de proyectos.

Personalice cada producto de trabajo para que se adecue a las necesidades del proyecto. Para consideraciones sobre la personalización, consulte el apartado de Personalización de la página de descripción de los productos de trabajo, o los pasos descritos debajo de la cabecera "Personalizar productos de trabajo por disciplina" en Tarea: Desarrollar guión de desarrollo.
