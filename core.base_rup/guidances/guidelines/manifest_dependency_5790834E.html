Directriz: Dependencia de manifiesto
En esta directriz se describe cómo utilizar una dependencia de manifiesto para correlacionar elementos de diseño con elementos de implementación.
Relaciones
Elementos relacionados
  • Modelo de implementación
Descripción principal

Explicación

Implementar dependencia se ha renombrado como Dependencia de manifiesto. Consulte Diferencias entre UML 1.x y UML 2.0 para obtener más información.

La dependencia de "manifiesto" se utiliza habitualmente para modelar el archivo o archivos que se utilizan para implementar un elemento de diseño. Esto puede incluir archivos de origen, o archivos ejecutables derivados.

Utilización

El uso de la dependencia "manifiesto" para relacionar explícitamente los elementos de implementación para diseñar elementos es opcional. Los convenios de denominación a menudo hacen esta correlación suficientemente clara. Por ejemplo, un convenio común es para un elemento de diseño y el archivo que lo implementa para que tengan el mismo nombre.
