Resumen

Las aplicaciones web se hacen cada vez más complejas y esenciales para la misión. Para facilitar la gestión de esta complejidad, deben modelarse. UML es el lenguaje estándar para el modelado de sistemas de software intensivo. Al modelar aplicaciones web con UML, queda claro que algunos de sus componentes no se ajustan bien a los elementos de modelado de UML estándar. Para ajustarse a una notación de modelado para todo el sistema (componentes web y componentes de nivel medio tradicionales), UML debe ampliarse. Este documento técnico presenta una ampliación de UML mediante el mecanismo de ampliación formal. La ampliación se diseña con el fin de que los componentes específicos de web puedan integrarse con el resto del modelo del sistema y para mostrar el nivel apropiado de abstracción y detalle adecuado para diseñadores, implementadores y arquitectos de aplicaciones web.
