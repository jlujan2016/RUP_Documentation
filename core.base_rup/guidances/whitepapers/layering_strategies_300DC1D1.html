Resumen

Existen diferentes técnicas para descomponer sistemas de software. Las capas son un ejemplo que se describe en este documento técnico. Estas técnicas tratan dos preocupaciones principales: la mayoría de los sistemas son demasiado complejos para comprenderlos en su totalidad, y son necesarias diferentes perspectivas de un sistema para usuarios diferentes.

Las capas se han adoptado en numerosos sistemas de software y en muchos textos, también en Rational Unified Process (RUP). No obstante, el concepto de capas no suele entenderse bien y se aplica incorrectamente. Este documento técnico aclara este concepto y describe el impacto de la aplicación de diferentes estrategias de capas.
