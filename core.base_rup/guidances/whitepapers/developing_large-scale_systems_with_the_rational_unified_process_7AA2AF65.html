Resumen

Al desarrollar sistemas de gran escala, la complejidad aumenta considerablemente. No basta con comprender un conjunto de productos de trabajo más complejo, sino que se introducen gastos generales ya que es necesario gestionar un conjunto de recursos más amplio. Este documento técnico describe un patrón de arquitectura que se utiliza para ayudar a controlar los gastos generales de la complejidad añadida. El patrón de arquitectura se denomina sistema de sistemas interconectados.

Este documento técnico introduce un patrón de arquitectura para sistemas de sistemas interconectados. Esta construcción permite la recursividad no sólo dentro del modelo, sino que considera cada subsistema como un sistema en sí mismo, y la recursividad se produce entre todos los conjuntos de productos de trabajo de cada uno de los sistemas. La arquitectura presentada se utiliza para sistemas que se implementan mediante varios sistemas de comunicación. Cada sistema implicado se describe según su propio conjunto de modelos, independientemente de otros modelos de sistemas.

Los ejemplos mostrados en este documento técnico ilustran que la arquitectura para modelar sistemas de sistemas interconectados es útil en muchas áreas de aplicaciones diferentes. De hecho, puede utilizar la arquitectura sugerida para cualquier sistema donde es posible ver las diferentes partes como sistemas en sí mismos.
