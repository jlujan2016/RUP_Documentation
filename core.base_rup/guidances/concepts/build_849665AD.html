Descripción principal

Una compilación es una versión operativa de un sistema o un componente de un sistema que demuestra un subconjunto de las posibilidades que se proporcionan en el producto final.

Las compilaciones son un componente integral del ciclo vital repetitivo; representan intentos continuos de demostrar las funciones desarrolladas hasta la fecha. Cada compilación se coloca en un control de configuración en caso de que sea necesario retrotraerse a una versión anterior cuando las funciones añadidas provocan daños o comprometen la integridad de la compilación.

Durante el desarrollo repetitivo de software, se producirán muchas compilaciones. Las compilaciones sirven para proporcionar puntos de revisión anticipados y ayudan a desvelar problemas de integración en el momento en que se introducen.
