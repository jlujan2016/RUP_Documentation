Concepto: Objetivo de capacidad operativa inicial
En esta directriz se tratan los criterios de evaluación para el objetivo de capacidad operativa inicial al final de l a fase de construcción. También se describe el estado de los artefactos esenciales.
Descripción principal

En el objetivo de capacidad operativa inicial, el producto está preparado para entregarse al equipo de transición. Se han desarrollado todas las funciones y se han completado todas las pruebas de versión alfa (si existe alguna). Además del software, se ha desarrollado un manual de usuario y hay una descripción del release actual.

Criterios de evaluación

Los criterios de evaluación de la fase de construcción incluyen las respuestas a las preguntas siguientes:

  • ¿El release de este producto es estable y está lo suficientemente maduro para desplegarse en la comunidad del usuario?
  • ¿Todos los interesados están preparados para la transición a la comunidad del usuario?
  • ¿Los gastos reales en recursos son aceptables en comparación con los gastos planificados?

Si el proyecto no alcanza este objetivo, es posible que la transición se posponga en un release.

Artefactos

Artefactos esenciales (en orden de importancia) Estado en el objetivo
"El sistema" El propio sistema ejecutable, listo para empezar la prueba de "versión beta".
Plan de despliegue La versión inicial desarrollada, revisada y con línea base. En proyectos más pequeños, puede incorporarse en el Plan de desarrollo de software.
Modelo de implementación (y todos los artefactos constitutivos, incluidos Elementos de implementación) Ampliado a partir del creado durante la fase de elaboración; todos los elementos de implementación creados al final de la fase de construcción.
Conjunto de aplicaciones de prueba ("prueba aleatoria") Pruebas implementadas y ejecutadas para validar la estabilidad de la compilación para los releases ejecutables que se crearon durante la fase de construcción.
Material de soporte para el usuario Manuales de usuario y otros materiales de formación. Borrador preliminar, basado en guiones de uso.  Puede ser necesario si el sistema tiene un aspecto de interfaz de usuario fuerte.
Plan de iteración Se ha completado y revisado el plan de iteración para la fase de transición.
Modelo de diseño (y todos los artefactos constitutivos) Actualizado con elementos de diseño nuevos identificados durante la terminación de todos los requisitos.
Proceso de desarrollo

El proceso de desarrollo, incluidos el guión de desarrollo y las directrices y plantillas específicas del proyecto, se ha perfeccionado en función de la experiencia del proyecto y se define lo suficiente para que continúe la siguiente fase.

Infraestructura de desarrollo

El entorno de desarrollo para la transición está en su lugar, incluidas todas las herramientas y el proceso de automatización para el proceso.

Modelo de datos Actualizado con todos los elementos necesarios para dar soporte a la implementación de la permanencia (por ej. tablas, índices, correlaciones objeto-relacional, etc.)
Artefactos opcionales Estado en el objetivo
Especificaciones suplementarias Actualizadas con requisitos nuevos (si hay alguno) descubiertos durante la fase de construcción.
Modelo de guión de uso (Actores, Guiones de uso) Actualizado con guiones de uso nuevos (si es que hay alguno) descubiertos durante la fase de construcción.
