Rol: Ingeniero de proceso
Este es un rol de soporte en RUP. El objetivo principal es proveer al equipo de proyecto con un proceso de desarrollo apropiado, y asegurar que los miembros del equipo no tienen dificultades para realizar su trabajo.

Descripción principal

Los ingenieros de proceso desempeñan un papel importante en cualquier equipo de gestión de un proyecto de software. El rol es responsable de todos los aspectos relacionados con el proceso del proyecto, como por ejemplo:

  • Personalizar el proceso para que coincidan las necesidades específicas del proyecto.
  • Educar y formar a los miembros del proyecto sobre cuestiones relacionadas con el proceso.
  • Garantizar que se cosecha una experiencia valiosa en el proyecto y que se reaprovecha dentro del proceso.
  • Asistir al gestor de proyectos en la planificación del proyecto.

Personal
Habilidades

Es importante que la persona que desempeña el rol ingeniero de proceso tenga una comprensión profunda de la definición subyacente del proceso para poder realizar elecciones informadas sobre su configuración. Idealmente, una persona que desempeñe este rol debe haber tenido múltiples experiencias trabajando en proyectos de software que utilicen RUP como proceso de desarrollo de software y deben comprender las diferentes consideraciones que se deben realizar al personalizar RUP para que se adapte al contexto de un proyecto de software específico.

Las habilidades y conocimientos apropiados para el rol ingeniero de proceso incluyen:

  • Conocimiento directo de la estructura de RUP y del conjunto de herramientas de soporte para crear procesos personalizados.
  • Buenas habilidades de comunicación para poder presentar el proceso a los equipos y animar a los miembros del equipo para que proporcionen comentarios para mejorar el proceso.
Propuestas de asignación

El rol ingeniero de proceso se puede asignar de los modos siguientes:

  • Asignar a la persona que desempeña el rol de gestor de proyectos al rol ingeniero de proceso. Este es un enfoque común para equipos pequeños y para equipos donde no hay recursos de ingeniería de proceso dedicados disponibles.
  • Para organizaciones más grandes, las personas que desempeñan este rol a menudo forman parte del grupo de proceso que definen las configuraciones organizativas y actúan como mentores del proceso en los proyectos.
  • Es una práctica común de las organizaciones nuevas en RUP y del desarrollo iterativo que busquen los recursos fuera de la empresa para desempeñar este rol en los primeros proyectos dirigidos por RUP.
