Tarea: Desarrollar el plan de aceptación del producto
En esta tarea se describe cómo crear un procedimiento escrito que han acordado el cliente y el equipo del proyecto para determinar la aceptabilidad de los entregables del proyecto y el proceso mediante el cual se resolverán los problemas identificados durante la aceptación del producto.

Descripción principal

La aceptación final de los entregables de un proyecto por parte del cliente es a menudo el origen de una cierta fricción en los proyectos de software. Normalmente es el resultado de una discrepancia entre la visión que tienen los clientes sobre cómo debería funcionar el producto y los métodos utilizados para valorar si los productos cumplen los requisitos especificados.

El cliente y el equipo del proyecto pueden evitar esta situación redactando conjuntamente un plan de aceptación del producto durante la fase inicial, en el que acuerdan un proceso predefinido y un conjunto de criterios mediante los cuales se valorará la aceptación del producto. Este plan ayuda al equipo del proyecto a crear un producto que el cliente pueda aceptar y ayuda a establecer las expectativas del cliente sobre cómo debe funcionar el producto. El plan de aceptación del producto también especifica cómo se resolverán los problemas que identifique el cliente durante la aceptación del producto.

Pasos
Definir responsabilidades del cliente y del equipo del proyecto

El primer paso consiste en identificar explícitamente qué partes del proceso de aceptación del producto serán responsabilidad del cliente y cuáles serán responsabilidad del equipo del proyecto. También debe identificar explícitamente la persona o grupo que tomará la decisión final de aceptación. Las responsabilidades pueden incluir las siguientes:

  • Entrega/recogida de software y documentación
  • Instalación de plataformas de prueba de hardware/software
  • Suministro de datos de prueba
  • Suministro de recursos para llevar a cabo las pruebas de aceptación
  • Respuesta puntual de los resultados de las pruebas de aceptación
Documentar los criterios de aceptación del producto

Los criterios de aceptación del producto se definen y acuerdan en la Tarea: Iniciar el proyecto durante la fase inicial y deben capturarse en el plan de aceptación del producto en ese momento. Durante la fase de elaboración, los criterios pueden desarrollarse de forma más detallada cuando se puedan identificar revisiones de prueba específicas.

Estos criterios se deben desarrollar conjuntamente entre la organización del cliente y el equipo del proyecto, y pueden incluir lo siguiente:

  • Entrega de todos los productos de trabajo identificados como entregables al cliente
  • Lista de participantes necesarios para la prueba de aceptación
  • Ubicaciones de prueba necesarias
  • Finalización satisfactoria de las evaluaciones del producto de trabajo identificadas en el plan de aceptación del producto
  • Finalización satisfactoria de la formación del cliente
  • Finalización satisfactoria de la instalación in situ
  • Medidas que identificarán hasta qué punto se han cumplido las especificaciones originales del proyecto
  • Medidas que identificarán hasta qué punto se han cumplido los objetivos del caso de negocio
Identificar artefactos y métodos de evaluación

A continuación, identifique qué productos de trabajo del proyecto deben entregarse al cliente para su aceptación. Para cada uno de ellos, debe identificar el método o métodos de evaluación que se utilizarán para asegurar que el producto de trabajo cumple los criterios de aceptación especificados. Más adelante en el proyecto, se desarrollarán listas de comprobación de revisión y casos de prueba detallados para proporcionar instrucciones paso a paso sobre cómo se llevarán a cabo estas evaluaciones.

Identificar los recursos necesarios

Una vez que se han identificado el número y el tipo de evaluaciones del producto de trabajo, identifique en el plan todos los recursos necesarios para llevar a cabo la tarea de aceptación del producto. En la lista de recursos, debe incluir los siguientes:

  • Personal
  • Hardware de sistema
  • Software
  • Datos
  • Documentación
  • Equipos especializados
Definir la planificación de aceptación del producto

Otro problema habitual con el proceso de aceptación del producto es cuando el cliente no establece suficiente prioridad en la tarea de aceptación, con el resultado de que el proceso se va arrastrando durante un largo período de tiempo. Es una buena idea incluir en el plan de aceptación del producto una planificación que detalle cuándo deben realizarse las diversas tareas de evaluación de la aceptación. Esta planificación se "incorporará" en la planificación del proyecto maestro dentro del plan de desarrollo de software.

Definir el proceso de resolución de problemas

Este paso final también es muy importante. En caso de que surjan problemas durante las evaluaciones de aceptación, es una buena idea acordar un proceso a seguir. Normalmente, este proceso simplemente seguiría el proceso de resolución de problemas del proyecto tal como se ha definido en el plan de resolución de problemas. Sin embargo, también es útil incluir aspectos tales como llegar a un acuerdo sobre cuándo un problema es real, la provisión de fondos para trabajos adicionales del equipo del proyecto o las penalizaciones contractuales. Llegar a un acuerdo de antemano sobre todos estos temas con el cliente, le facilitará considerablemente la finalización del proyecto.
