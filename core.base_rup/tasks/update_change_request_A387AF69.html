Tarea: Actualizar solicitud de cambio
En esta tarea se define quién puede realizar cambios en una solicitud de cambio y cómo hacerlo.
Disciplinas: Configuración y Gestión de cambios
Objetivo
  • Un administrador de CCB designado o el propietario asignado de una solicitud de cambio puede realizar actualizaciones en el formulario de CR para indicar su estado actual así como cualquier otra información pertinente.

Pasos
Recuperar el formulario de solicitud de cambio

El formulario de solicitud de cambio es un producto de trabajo enviado formalmente que se utiliza para realizar el seguimiento de todas las solicitudes durante todo el ciclo de vida del proyecto. Incluye las nuevas características, las solicitudes de mejora, los defectos, los requisitos modificados, etc. El historial de cambios se mantendrá junto con la CR, incluidos todos los cambios de estado, y las fechas y los motivos de dicho cambio. Esta información estará disponible para las revisiones repetidas y para el cierre final. En Producto de trabajo: Solicitudes de cambio se incluye un formulario de solicitud de cambio de ejemplo.

Actualizar y volver a enviar la solicitud de cambio

Si es necesaria más información (Más información) para evaluar una CR, o si una CR se rechaza en un punto del proceso (p.ej., se confirma como Duplicada, Rechazada, etc.), se notifica al emisor para que pueda actualizar la CR con información nueva. Entonces la CR actualizada se vuelve a enviar a la cola de revisión de CCB para tener en cuenta los nuevos datos.

Los estados típicos que puede pasar una solicitud de cambio se muestran en Gestión de solicitudes de cambio.
