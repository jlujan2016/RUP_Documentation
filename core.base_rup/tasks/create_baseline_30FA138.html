Descripción principal

Entre los buenos candidatos para una línea base se encuentran los conjuntos de archivos y directorios bajo control de versión que se desarrollan, integran y liberan juntos. Una línea base identifica una y sólo una versión de un elemento. Las líneas base compuestas son agrupaciones de líneas base entregadas por equipos que trabajan en subsistemas distintos del proyecto global.

Parte de la metodología de desarrollo iterativo consiste en crear constantemente un producto ejecutable. Una línea base debe considerarse como el frente motor de desarrollo que contiene los productos de trabajo necesarios para el desarrollo posterior. Por tanto, las líneas base deben crearse al final de cada iteración.

Las prácticas de creación de líneas base de proyecto se llevan a cabo de acuerdo con el paso de identificación de la configuración, creación de líneas base y archivado definido en la Tarea: Establecer políticas CM.
