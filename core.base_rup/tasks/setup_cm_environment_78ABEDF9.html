Tarea: Configurar el entorno de gestión de la configuración (CM)
En esta tarea se describe cómo crear un entorno donde se pueda desarrollar y crear el producto.
Disciplinas: Configuración y Gestión de cambios
Objetivo

El objetivo de esta tarea es establecer un entorno donde se pueda desarrollar y crear el producto. Esto se realiza en dos partes: primero se establece el entorno de hardware y, a continuación, se establece el entorno de desarrollo.

Descripción principal

Configurar el entorno de CM implica asignar los recursos de la máquina (servidores y espacio de disco) e instalar las herramientas de gestión de la configuración.

Configurar el entorno de desarrollo implica crear depósitos, configurar la estructura de directorios del producto e importar los archivos existentes. El entorno inicial sirve como línea base para los próximos trabajos de desarrollo.

Pasos
Configurar el entorno de hardware de CM
Objetivo:  Asignar los recursos de hardware necesarios para instalar y configurar la Herramienta CM. 

El gestor de configuración trabaja con el administrador del sistema para asignar recursos de la máquina e instalar las herramientas de software necesarias.

Las consideraciones clave (en orden de prioridad) de la máquina dedicada para ejecutar el servidor que arbitra el acceso a los datos reales en el depósito del proyecto son los siguientes:

  • Requisitos de memoria
  • Requisitos de entrada / salida de disco
  • Ancho de banda de red
  • Espacio de disco de depósito del proyecto

Para obtener más información sobre cada uno de estos elementos, consulte Producto de trabajo: Depósito de proyecto

Correlacionar la arquitectura con el depósito
Objetivo:  La estructura de directorios del producto está organizada de forma lógica para garantizar que exista un marcador de posición para todos los productos de trabajo relacionados con el proyecto.  

La estructura de directorios del producto proporciona un marcador de posición anidado lógicamente para todos los productos de trabajo relacionados con el producto. La forma del directorio (que sirve como depósito del proyecto) depende del número de subsistemas en el sistema general y del número de elementos en cada subsistema.

La estructura lógica del producto no emerge hasta que estas tareas están en proceso: Análisis y diseño. Se debe crear un depósito de proyecto inicial para la gestión y la planificación de productos de trabajo. En Concepto: Estructura de directorios del producto se proporcionan instrucciones para la estructura inicial.

El resto de la estructura se puede elaborar una vez tomadas las decisiones de diseño, cuando la naturaleza de la vista de implementación muestra más claramente cómo se van a empaquetar los distintos elementos de diseño para la implementación.

Cree un marcador de posición para cada subsistema que se tenga que implementar en la estructura de directorios. Calcule los requisitos de almacenamiento de los productos de trabajo que se van a desarrollar y asegúrese de que haya suficiente almacenamiento físico. A efectos de CM, debe haber un alto grado de cohesión entre los elementos internos de la estructura de directorios del producto. Los subsistemas deben tener interfaces claramente definidas con las otras partes del sistema, y se deben poder construir y probar de manera independiente. El objetivo clave es permitir el desarrollo paralelo e independiente de los sistemas por parte de distintos equipos. La idea es acelerar significativamente el desarrollo, promover la reutilización y facilitar el mantenimiento del sistema.

Crear el conjunto inicial de elementos versionados
Objetivo:  Crear una línea base inicial de productos de trabajo del proyecto.  

Incluso en los proyectos sin gestión de la configuración existe el concepto de una estructura de directorios y un conjunto de material existente que se utiliza en el proyecto de forma continua. La idea es exportar/importar el material existente a la estructura creada para el desarrollo del producto.

Definir niveles de promoción de línea base
Objetivo:  Garantizar que todos los elementos almacenados en el depósito del proyecto compartan un conjunto común de niveles de promoción "legales".  

Una línea base es una única versión del depósito del proyecto. La calidad o el estado de esa línea base vienen indicados por un nivel de promoción de línea base. Todos los elementos almacenados en el depósito del proyecto comparten un conjunto común de niveles de promoción "legales", preferiblemente con definiciones coherentes entre varios proyectos. Consulte el apartado Gestión de la configuración en Concepto: Línea base para obtener más información.
