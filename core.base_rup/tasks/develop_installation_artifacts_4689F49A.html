Descripción principal

Al desarrollar productos de trabajo de instalación, el implementador debe considerar cómo debe empaquetarse, personalizarse e instalarse el producto.

Un conjunto típico de productos de trabajo de instalación incluye:

  • Scripts de instalación
  • Archivos de configuración
  • Instrucciones de instalación

Los productos de trabajo de instalación deben desarrollarse con el mismo rigor y proceso que se siguen para desarrollar cualquier otro producto de trabajo. Normalmente hay un requisito de un conjunto distinto de productos de trabajo de instalación para cada plataforma en la que debe instalarse el producto.

El caso de ejemplo de instalación más simple es el de un usuario individual que instala el software en un único sistema personal (nodo). Sin embargo, en talleres de desarrollo más grandes, normalmente un único rol (administrador de red) tiene la responsabilidad de las tareas globales de gestión de software a nivel de empresa. En este caso, supone una gran ventaja poder instalar y actualizar el software de todas las estaciones de trabajo de forma rápida y sencilla desde una ubicación. El precio de esta comodidad se paga con la complejidad del software de instalación necesario para distribuir el software y personalizar las estaciones de trabajo desde un único servidor o en toda la empresa. 

Es posible que el desarrollador desee utilizar algunos paquetes comercialmente disponibles que proporcionan plantillas y objetos de interfaz preincorporados que facilitan el empaquetado del software para su instalación. Estos paquetes pueden personalizarse para mostrar imágenes del producto y logotipos corporativos.
