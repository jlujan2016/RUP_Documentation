Tarea: Crear espacios de trabajo de desarrollo
En esta tarea se describe cómo crear y utilizar un espacio de trabajo de desarrollo.
Disciplinas: Configuración y Gestión de cambios
Objetivo
  •  Un espacio de trabajo de desarrollo es un área de desarrollo privado que proporciona un entorno en el que un miembro del equipo puede realizar cambios en los productos de trabajo sin que los cambios sean visibles inmediatamente para los demás miembros del equipo.
  • Parte del proceso de creación de un espacio de trabajo es componer una vista que se configura para garantizar que cualquier miembro del equipo puede 'llegar' a la versión necesaria de cualquier producto de trabajo dado que necesitan para realizar su trabajo. Las políticas de proyecto determinan qué productos de trabajo son visibles o modificables y por quién.

Descripción principal

Ser miembro de un equipo del proyecto es saber en qué se puede trabajar y dónde se debe entregar el trabajo para que éste se pueda integrar y probar con el trabajo realizado por otros miembros del equipo. Un proyecto normalmente tiene un área de trabajo compartida y varias áreas de desarrollo privado. El espacio de trabajo de desarrollo privado permite al desarrollador trabajar en tareas de forma aislada.

Cuando un desarrollador se une a un proyecto tiene acceso a determinados productos de trabajo y a un espacio de trabajo de integración.El paso siguiente es que los desarrolladores creen un espacio de trabajo de desarrollo privado y lo llenen con el contenido de la línea base del proyecto como punto de partida. El gestor de configuración crea la línea base del proyecto al configurar el entorno CM del proyecto.

En el espacio de trabajo de desarrollo, los desarrolladores realizan tareas para trabajar en productos de trabajo. El pedido de trabajo proporciona una descripción de lo que debe hacer un miembro del equipo. Puede compartir su trabajo con otros miembros del equipo del proyecto entregando el conjunto cambiado de productos de trabajo en el área de trabajo compartido del proyecto.

En el área de trabajo compartido, el trabajo individual se integra con el de otros miembros del equipo. Periódicamente, se crea una nueva línea base en el área de trabajo compartido que incorpora el trabajo entregado.

Los desarrolladores actualizan sus espacios de trabajo de desarrollo privados para incluir el conjunto de versiones representadas por la nueva línea base. Periódicamente, a medida que mejoran la calidad y la estabilidad de las líneas base, el atributo de nivel de promoción de las líneas base cambia para reflejar el nivel apropiado de madurez. Los atributos de la línea base de ejemplo son built, tested o release; estos atributos se determinan como parte de las políticas CM del proyecto.
