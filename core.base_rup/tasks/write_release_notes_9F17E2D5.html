Tarea: Escribir las notas del release
En esta tarea se describe cómo proporcionar notas del release que describen las nuevas características y los cambios más importantes realizados en el release, así como los errores y limitaciones conocidos o las soluciones provisionales al utilizar el producto.

Descripción principal

Las notas del release proporcionan un resumen de los cambios realizados desde el último release. También documentan los errores y limitaciones conocidos o las soluciones provisionales al utilizar el producto. Las notas del release se incluyen en la unidad de despliegue o en el producto.

Una buena práctica consiste en crear las notas del release para la valoración de iteración al final de cada iteración. Sin embargo, las notas del release pueden actualizarse y mantenerse para cada compilación y, a continuación, actualizarse para el release formal del producto.
