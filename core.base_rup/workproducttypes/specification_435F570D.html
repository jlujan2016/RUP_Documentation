Clase de producto de trabajo: Especificación
Estos productos de trabajo definen las interfaces entre las diferentes partes del proyecto, especialmente entre diferentes dominios. Definen con precisión lo que un elemento va a hacer o debe hacer. Por ejemplo, la Especificación de requisitos de software define cómo se llevan a cabo los objetivos empresariales con requisitos del sistema; el Documento de arquitectura de software muestra cómo se correlacionan los requisitos del sistema con los aspectos clave del diseño.
