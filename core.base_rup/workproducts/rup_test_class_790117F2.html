Artefacto: Clase de comprobabilidad
Este artefacto es una clase especializada en el modelo de diseño que representa el comportamiento específico de prueba que el software soportará.
Clases de producto de trabajo: Elemento de modelo
Objetivo

El objetivo de la clase de comprobabilidad es capturar el diseño de la funcionalidad específica de prueba necesaria para facilitar las pruebas. Esta funcionalidad específica de prueba debe incorporarse en el modelo de diseño de software para que se pueda tener en cuenta en el diseño de software completo. Existen varios tipos de comportamientos específicos en la prueba, dos de los cuales son:

  • Los "fragmentos para simulación" para clases de diseño que necesitará para simular o si ha decidido no incluir versiones completadas de software que se utilizarán como destino de prueba.
  • Interfaces especializadas o salida, que proporcionan la visibilidad o control necesarios para efectuar las pruebas.
