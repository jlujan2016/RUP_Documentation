Artefacto: Ejecución de casos de uso
Este artefacto describe cómo se ejecuta un guión de uso concreto dentro del modelo de diseño, en términos de objetos de colaboración.
Clases de producto de trabajo: Elemento de modelo
Objetivo

El objetivo de la ejecución de guiones de uso es separar las preocupaciones de los especificadores del sistema (tal como se representan en el modelo de guión de uso y los requisitos del sistema) desde las preocupaciones de los diseñadores del sistema. La ejecución de guiones de uso proporciona una construcción en el modelo de diseño que organiza los productos de trabajo relacionados con un guión de uso pero que pertenezcan al modelo de diseño. Estos productos de trabajo relacionados consisten, habitualmente, en la comunicación y los diagramas de secuencia que expresan el comportamiento del guión de uso en términos de objetos de colaboración.

Personalización
Opciones de representaciónRepresentación UML:

Colaboración o CollaborationInstanceSet, estereotipada como <<realización de guión de uso>>. 

Las realizaciones expresan el comportamiento de un conjunto de elementos de modelo que realizan parte o todo un Producto de trabajo: Guión de uso. Como resultado, debería haber una ejecución de guiones de uso que se debe expresar en el modelo de diseño. De forma similar, si los guiones de uso no se utilizan, las ejecuciones de guiones de uso también se omitirán.
