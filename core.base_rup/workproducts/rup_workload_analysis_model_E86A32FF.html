Artefacto: Modelo de análisis de carga de trabajo
Este artefacto es un modelo que identifica uno o más perfiles de carga de trabajo estimados para que definan con precisión el estado del sistema de interés donde se pueda llevar a cabo la evaluación del software y/o su entorno operativo.
Dominios: Prueba
Clases de producto de trabajo: Modelo
Amplía: Caso de prueba
Objetivo
  • Para definir con precisión las condiciones de carga bajo las cuales los elementos del destino de la prueba deben operar en el entorno de destino de configuración.
  • Para definir una carga de trabajo realista representativa que permita valorar con precisión el riesgo de rendimiento.

Descripción
Descripción principalLos perfiles de carga de trabajo que componen el modelo de análisis de carga de trabajo  representan las condiciones necesarias que deben simularse contra los elementos del destino de la prueba debajo de una o más configuraciones de entorno de prueba. Estas se determinan habitualmente analizando las características anticipadas o existentes del actor, las estadísticas de la empresa del usuario (casos de uso), etc.
Esquematización breve

1. Introducción

Identifica la finalidad, los antecedentes y los objetivos de la prueba de rendimiento dentro de este proyecto.

2. Atributos y variables del sistema

Identifica los atributos y las variables del sistema que identifican de manera exclusiva la carga de trabajo del sistema que se está modelando.

4. Definiciones de actor

Identifica las clases de clientes externos cuyos casos de ejemplo de caso de uso deberán modelarse para simular / emular las cargas en el sistema que se somete a prueba. Además, esta sección identifica la proporción que supone para cualquier actor la carga de una prueba de rendimiento.

7. Atributos del actor

Identifica los atributos y las variables de cada actor que identifican de manera exclusiva las diferentes características de los clientes externos del sistema. Para cada actor, identifica información como por ejemplo humano o no humano, velocidad de entrada de datos, tiempo de pensamiento, estilo de la transacción, complejidad de la transacción y patrones de comportamiento que caracterizan la variabilidad de la interacción del usuario con el sistema.

6. Perfil de trabajo del actor

Identifica los casos de ejemplo de caso de uso que ejecuta un actor y el porcentaje de tiempo o proporción de esfuerzo que emplea el actor que ejecuta los casos de ejemplo de caso de uso para llevar a cabo sus responsabilidades empresariales totales.

3. Perfil de carga de trabajo

Para un determinado perfil, identifica el número de clientes externos que se simulan / emulan durante la prueba, incluyendo el número, tipo y distribución de las transacciones. Un perfil puede definirse en términos de "carga máxima", "carga media", etc.

5. Medidas y criterios

Identifica las medidas y los criterios que deben utilizarse para evaluar la consecución satisfactoria de los objetivos de rendimiento identificados. Las medidas normalmente incluyen límites de tiempos de respuesta o capacidad de rendimiento.

8. Requisitos de emulación de terminal remoto

Identifica los requisitos y las restricciones que deben tenerse en cuenta a la hora de crear una configuración de entorno de prueba que sea aceptable para implementar y ejecutar las pruebas de rendimiento.

Personalización
Opciones de representación

Es posible que se deba modificar el modelo de análisis de carga de trabajo (contenido y formato) para que satisfaga las necesidades de estándares internos o externos, directrices, etc. Opcionalmente, algunos aspectos del modelo de análisis de carga de trabajo pueden encapsularse en el plan de prueba
