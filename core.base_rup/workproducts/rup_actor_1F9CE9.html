Artefacto: Actor
Este artefacto define un conjunto coherente de roles que los usuarios del sistema pueden desempeñar cuando interactúan con este. Una instancia de este artefacto se puede llevar a cabo en un sistema individual o externo.
Clases de producto de trabajo: Elemento de modelo
Amplía: Requisito de software
Objetivo

Los diferentes interesados utilizan este producto de trabajo con diferentes objetivos:

  • Analistas de sistemas: para definir los límites del sistema.
  • Diseñadores de interfaz de usuario: para capturar las características en actores humanos.
  • Autores de caso de uso : para describir los casos de uso y su interacción con los actores.
  • Analistas de objeto: para realizar casos de uso y su interacción con los actores.

Descripción
Esquematización breve

Al crear este artefacto, debe tener en cuenta varios aspectos del artefacto, incluidos los siguientes:

  • los distintos grupos de interés o interesados que pueden tener requisitos para contribuir
  • los distintos tipos de requisitos (categorías, dimensiones) que deben tenerse en cuenta

Personalización
Opciones de representaciónRepresentación UML: Actor

El actor puede tener las siguientes propiedades:

  • Nombre: El nombre de un actor.
  • Descripción breve: Breve descripción de la esfera de responsabilidad del autor y de para qué el actor necesita el sistema. 
  • Características: Para los actores humanos: El entorno físico del actor, el número de usuarios que representa el actor, el nivel de conocimientos del dominio del actor, el nivel de experiencia informática del actor, otras aplicaciones que utiliza el actor y otras características generales como por ejemplo el género, la edad o el bagaje cultural. 
  • Relaciones: Relaciones, como las generalizaciones de actores, y asociaciones de comunicación en las que participa el actor. 
  • Diagramas: Diagramas locales para el actor, como los diagramas de casos de uso que describan las asociaciones de comunicación con los casos de uso . 

Decida qué propiedades va a utilizar y cómo va a utilizarlas. Concretamente, debe decidir con qué nivel de detalle debe describirse la propiedad "Características".
