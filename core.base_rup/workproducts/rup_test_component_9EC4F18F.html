Artefacto: Elemento de comprobabilidad
Este artefacto es un elemento de implementación especializado que realiza el comportamiento específico de la prueba al que da soporte el software.
Clases de producto de trabajo: Solución
Objetivo

El objetivo del Elemento de comprobabilidad es implementar funcionalidades específicas de la prueba que faciliten la prueba, ya sean manuales o automáticas. Existen varios tipos de comportamientos específicos en la prueba, dos de los cuales son:

  • Los "fragmentos para simulación" para elementos de implementación que necesitará para simular o si ha decidido no incluir completamente en un destino de prueba.
  • Los elementos que proporcionan interfaces especializadas o salidas.

Factores clave

Los elementos de comprobabilidad se crean habitualmente y se modifican en paralelo con la creación y la modificación de la aplicación o del sistema correspondiente a los elementos de implementación.
