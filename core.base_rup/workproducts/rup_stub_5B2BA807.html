Artefacto: Prueba de fragmento para simulación
Este artefacto es un elemento de implementación especializado que se utiliza para efectuar pruebas, que simula un componente verdadero.
Clases de producto de trabajo: Solución
Amplía: Elemento de comprobabilidad
Objetivo

El objetivo de la prueba de fragmento para simulación es simular un componente real que no está disponible para la prueba.

Descripción
Esquematización breve

Existen dos aspectos respecto a la Prueba de fragmento para simulación: el grado de emulación y el ciclo vital. Visto el ámbito y los objetivos de la prueba, la implementación de un fragmento para simulación puede abarcar una clase casi vacía compatible con un conjunto de interfaces o bien un emulador avanzado que funcionará de modo muy parecido al componente real en cuanto a la funcionalidad se refiere. Excepto para las pruebas sencillas desechables, los fragmentos de simulación tendrán el mismo ciclo vital que los componentes que prueban y, por lo tanto, en la mayoría de los casos deben tratarse igual que el código del producto.

Factores clave

Los elementos de comprobabilidad se crean habitualmente y se modifican en paralelo con la creación y la modificación de la aplicación o del sistema correspondiente a los elementos de implementación.
