Descripción
Descripción principal

La lista de ideas de prueba proporciona una capa de abstracción entre el plan de prueba conceptual y el guión de prueba más detallado o el script de prueba concreto. Se utiliza para capturar ideas iniciales para pruebas potenciales, que a menudo están mal formadas, o ideas parciales sobre las que las pruebas se pueden razonar. Este producto de trabajo es especialmente útil antes en el ciclo de desarrollo o cuando da soporte a productos de trabajo del proyecto que no están disponibles o están incompletos.

Esquematización breve

Cada Lista de ideas de prueba debe identificarse considerando todas las perspectivas diferentes que sean posibles, incluyendo varias de las siguientes:

  • ¿Se han revisado todos los catálogos de ideas de prueba relevantes?
  • ¿Existe algún riesgo de calidad que no se haya representado en la lista?
  • ¿Existe algún modelo de anomalía no representado en la lista?
  • ¿Ha considerado la utilización de todos los posibles ataques?
  • ¿Ha considerado uno o más culebrones relevantes?
  • ¿Hay alguna otra idea que le parezca digna de ser considerada?
  • ¿Cuál es la importancia relativa de cada idea de la lista?

Cuando tenga la lista inicial de ideas, considere si existen ideas relacionadas en la lista que se podrían combinar o consolidar. Como norma general, la mayoría de listas deben contener siete entradas, más o menos dos. Para obtener más detalles sobre el contenido de una Lista de ideas de prueba, consulte las directrices que se listan en la sección Más información de la tabla de cabecera.

Personalización
Opciones de representación

En ciertos dominios y culturas de prueba, las ideas de prueba no se reconocen, o se consideran productos de trabajo informales. Como tales, el contenido y el formato de la Lista de ideas de prueba puede requerir modificaciones para cubrir las necesidades de cada organización y proyecto concretos.

Cuando se registran (formal o informalmente), se suelen utilizar dos estilos principales:

  • El primero es una estructura de documento de texto estándar que utiliza un formato similar al que se ha destacado anteriormente. Habitualmente, varias ideas de prueba se presentan juntas, mientras que una única idea de prueba sola no se considera que represente una lista suficiente.
  • El segundo utiliza alguna forma de tabla o de base de datos. Las ideas de prueba se especifican, una por línea con columnas proporcionadas para facilitar la ordenación y el filtrado por diferentes criterios. Las matrices de prueba o las tablas de causa y efecto se pueden considerar formas alternativas de las listas de ideas de prueba.

También debe tenerse en cuenta la medición continua de las ideas de prueba para el progreso, la eficacia, la gestión de cambios, etc. Considere la utilización de una cobertura de la prueba basada en especificaciones, donde cada idea de prueba o cada lista de ideas de prueba efectúe el rastreo hasta, como mínimo, una entrada de especificación que deba probarse. Por ejemplo, rastree hasta los elementos de especificación de requisitos que se deben probar que habitualmente reflejan algún subconjunto de los requisitos del producto total (consulte Técnica: Medidas clave de la prueba).

Opcionalmente, las ideas de prueba se pueden mantener como parte de un Guión de prueba o Script de prueba. También se puede hacer referencia a la lista desde, o en esfuerzos de prueba menores, incluidos dentro del Plan de prueba.
